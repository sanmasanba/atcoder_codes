# 048

## 問題リンク

[ABC310 E - NAND repeatedly](https://atcoder.jp/contests/abc310/tasks/abc310_e)

## キーワード

結合演算の部分列数は、演算結果ごとに右端までの個数を持つ

## 何に着目するか

各部分文字列に対して NAND を左から順に適用します。途中の演算結果は `0` か `1` の二通りしかありません。右端を固定したとき、「結果が 0 / 1 の部分文字列数」だけを持てば、開始位置を個別に管理する必要はありません。

## 解法方針

ここまでの右端で結果が `0`,`1` となる部分文字列数を `dp[0], dp[1]` とします。次のビットを `x` とすると、既存の全部分文字列に `x` を NAND し、さらに長さ 1 の新しい部分文字列 `x` を加えます。

NAND の遷移表は次です。

|前の値 `a`|`x=0` のとき `NAND(a,x)`|`x=1` のとき `NAND(a,x)`|
|---:|---:|---:|
|`0`|`1`|`1`|
|`1`|`1`|`0`|

したがって更新は次のようになります。

```text
x = 0: new_dp = [1, dp[0] + dp[1]]
x = 1: new_dp = [dp[1], dp[0] + 1]
```

各右端で新たに結果 1 となった数 `new_dp[1]` を答えに加えます。

## tips

### 実装

更新前の `dp` を使うため、`new0`, `new1` へ一度計算してから代入します。初期値は空部分文字列を数えないので `dp=[0,0]` です。

答えは部分文字列数で `O(N^2)` になり得るため、64 bit 整数を使います。

### よくある誤り

- 長さ 1 の部分文字列を加え忘れる。式中の `+1` がそれです。
- NAND を XOR / OR と混同する。`0 NAND 0` も `1` です。
- 更新済みの `dp[0]` を使って `dp[1]` を計算する。同時更新にします。

### 計算量

各文字に対して定数回の計算だけなので、時間 `O(N)`、追加メモリ `O(1)` です。
