# 026

## 問題リンク

[ABC213 F - Common Prefixes](https://atcoder.jp/contests/abc213/tasks/abc213_f)

## キーワード

隣接要素との最小値の総和は、単調スタックで寄与を数える

## 何に着目するか

各接尾辞について、全接尾辞との最長共通接頭辞（LCP）の和を求めます。接尾辞を文字列の開始位置順に並べても、似ている接尾辞は近くに集まりません。辞書順に並べる接尾辞配列を使うと、似た接尾辞が近くに集まります。

辞書順で位置 `i < j` の二つの接尾辞の LCP は、その間の LCP 配列の最小値です。従って問題は「各位置について、左右の区間最小値の総和」を求める問題になります。

## 解法方針

接尾辞配列を `SA`、隣接する接尾辞の LCP を `LCP` とします。

```text
SA 上の順番:    suffix[SA[0]]  suffix[SA[1]]  suffix[SA[2]]  ...
LCP:                         LCP[1]          LCP[2]

LCP(SA[i], SA[j]) = min(LCP[i+1], ..., LCP[j])   (i < j)
```

各接尾辞自身との LCP はその接尾辞の長さなので、最初に `answer[pos] = N - pos` を入れます。残りは、接尾辞配列上で左側にある相手からの寄与と右側にある相手からの寄与を別々に足します。

### 片側の寄与をスタックで求める

右へ 1 つ進むごとに、過去の各位置から現在位置までの区間最小値は、新しい `LCP` との `min` に置き換わります。値が単調増加になるよう `(値, 個数)` をスタックで管理します。

|新しい値 `x` を追加するとき|処理|
|---|---|
|スタック末尾の値 `> x`|取り出し、個数をまとめる。総和から `値×個数` を引く|
|取り出し後|`x×まとめた個数` を総和に足す|
|最後|`(x, 個数)` を積む。現在の総和が片側の寄与|

スタックから取り出した要素は二度と戻りません。そのため、全体での push/pop 回数は `O(N)` です。左から右への走査で「左側との LCP 和」、配列を反転した同じ走査で「右側との LCP 和」を得ます。

最後に、接尾辞配列上の位置 `rank` で得た値を、元の開始位置 `SA[rank]` に戻して出力します。

## tips

### 実装

C++ なら AtCoder Library の `suffix_array` と `lcp_array` を利用できます。LCP 和は最大で `O(N^2)` になるので、答え・スタック内の個数との積・総和は `long long` を使います。

片側を処理する関数を作り、`SA` と `LCP` を反転してもう一度呼ぶと、左右で別実装にするより添字のミスを減らせます。`LCP` の「どの二接尾辞の間の値か」は実装前に 0-indexed / 1-indexed の定義を固定します。

### よくある誤り

- LCP の最小値を区間和のように足す。二つの接尾辞の LCP は区間の**最小値**です。
- 自分自身との比較を忘れる。開始位置 `pos` の寄与は `N-pos` です。
- 同じ LCP 値を別々に積み続ける。個数を併せて持つことで、まとめて総和を更新できます。

### 計算量

SA・LCP 構築を除く集計は `O(N)` です。ACL の SA-IS を使えば全体 `O(N)`、比較ベースの接尾辞配列でも通常は `O(N log N)` 程度です。メモリは `O(N)` です。
