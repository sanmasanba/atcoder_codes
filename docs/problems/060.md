# 060

## 問題リンク

[ABC289 E - Swap Places](https://atcoder.jp/contests/abc289/tasks/abc289_e)

## キーワード

複数人が同時に動くなら、全員の位置の組を状態にして BFS する

## 何に着目するか

二人は毎手同時に一辺ずつ移動します。一人だけの位置を知っても、もう一人の色との条件を判定できません。状態を `(高橋の頂点, 青木の頂点)` の組で持つと、一回の同時移動が一つのグラフ辺になります。

全手のコストは等しいので、積グラフ上の最短手数は BFS で求められます。

## 解法方針

状態 `(u,v)` は「高橋が `u`、青木が `v` にいる」ことを表します。状態が有効であるのは `color[u] != color[v]` のときだけです。

`u→nu`, `v→nv` がそれぞれ元グラフの辺なら、次の同時移動があります。

```text
(u, v) ──> (nu, nv)
```

ただし遷移先の二頂点の色が同じなら、その状態はキューに入れません。始点 `(0,N-1)` から BFS し、目標 `(N-1,0)` の距離を出力します。

```text
元グラフの一手: u→nu と v→nv
積グラフの一手: (u,v)→(nu,nv)
```

同時に一手ずつ動くため、どの積グラフ辺もコスト 1 です。初期状態の色が同じなら、最初から動けず答えは `-1` です。

## tips

### 実装

距離は `dist[u][v]` の `N×N` 配列で持ち、未訪問を `-1` にします。キューから `(u,v)` を取り出し、`adj[u]` と `adj[v]` の二重ループで候補を作ります。

各テストケースごとにグラフ・距離配列を作り直します。

### よくある誤り

- 二人を交互に動かす通常の BFS にする。問題では毎手同時に動きます。
- 色条件を出発状態だけで確認する。毎回の遷移先でも異色でなければなりません。
- `(u,v)` と `(v,u)` を同じ状態としてまとめる。二人の役割が異なるため別状態です。

### 計算量

状態数は `O(N^2)` です。各状態で隣接頂点の組を列挙するため、最悪 `O(M^2)` 程度、距離配列のメモリは `O(N^2)` です。問題の全テストケース制約内で十分です。
