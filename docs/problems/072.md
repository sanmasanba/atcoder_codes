# 072

## 問題リンク

[ABC238 E - Range Sums](https://atcoder.jp/contests/abc238/tasks/abc238_e)

## キーワード

区間和の等式は、prefix 和どうしの同値関係に変換する

## 何に着目するか

未知の数列の prefix 和を `P[i]=A[1]+...+A[i]` とします。区間 `[l,r]` の和が 0 という条件は `P[r]=P[l-1]` です。つまり、二つの prefix 位置の値が等しいという同値関係になります。

## 解法方針

prefix 位置 `0..N` を頂点にします。条件 `[l,r]` ごとに、頂点 `l-1` と `r` を無向辺で結びます。最終的に必要なのは `P[0]=P[N]`、すなわち頂点 `0` と `N` が同じ連結成分にあることです。

```text
[l,r] の和が 0
      ↓
P[l-1] = P[r]
      ↓
頂点 l-1 と r を結ぶ
```

条件が連鎖すれば等式も連鎖します。Union-Find なら各条件を併合し、最後に `same(0,N)` を見るだけです。

## tips

### 実装

頂点数は元配列の長さではなく `N+1` です。入力が 1-indexed なので、左端だけ `-1` します。

### よくある誤り

- 頂点を元配列の要素番号にする。使うのは要素の間にある prefix 位置です。
- `[l,r]` を `l` と `r` で結ぶ。正しくは `l-1` と `r` です。
- 区間和そのものを計算しようとする。数列の値は与えられていません。

### 計算量

Union-Find を使うと時間は `O(Qα(N))`、メモリは `O(N)` です。
