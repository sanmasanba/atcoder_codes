# 067

## 問題リンク

[ABC227 D - Project Planning](https://atcoder.jp/contests/abc227/tasks/abc227_d)

## キーワード

資源を均等に配る最大個数は、一人あたりの上限を切って二分探索する

## 何に着目するか

`x` 個のプロジェクトを作ると固定します。一種類の資源を一プロジェクトへ何度も投入しても、そのプロジェクトでは一回分しか役に立ちません。よって資源 `A[i]` が `x` プロジェクトへ提供できる有効な量は高々 `min(A[i],x)` です。

この有効量の合計が `Kx` 以上かが、`x` 個作れるための必要十分条件です。

## 解法方針

判定関数を

```text
feasible(x) ⇔ Σ min(A[i], x) ≥ K × x
```

とします。左辺は各資源を最大一回ずつ各プロジェクトに配ったときの総提供可能量、右辺は必要な枠数です。

`x` が可能なら、それより少ないプロジェクト数も可能です。したがって `feasible(x)` は単調で、最大の真を二分探索できます。

|資源量 `a`|x プロジェクトへの最大寄与|
|---:|---:|
|`a < x`|`a`|
|`a ≥ x`|`x`|

上限は総資源量を `K` で割った `sum(A)//K` です。これを越えると、同じ資源を重複利用しないとしても総量が足りません。

## tips

### 実装

`ok=0`, `ng=sum(A)//K+1` のように「ok は可能、ng は不可能」を保つ二分探索にします。判定中の和は `K*x` 以上になったら打ち切ってもよいです。

`sum(A)` や `K*x` は 64 bit を超え得る言語ではより大きい型を使います。

### よくある誤り

- 判定を `sum(A)≥Kx` だけにする。一資源が同じプロジェクトの複数枠を埋めることはできません。
- `min(A[i],K)` と切る。上限は一プロジェクト数 `x` です。
- 最大値を求める二分探索で `ng` 側を可能値にする。

### 計算量

一回の判定は `O(N)`、二分探索回数は `O(log(sum(A)/K))` なので、時間 `O(N log sum(A))`、追加メモリ `O(1)` です。
