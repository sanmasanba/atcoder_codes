# 061

## 問題リンク

[ABC201 D - Game in Momotetsu World](https://atcoder.jp/contests/abc201/tasks/abc201_d)

## キーワード

二人零和ゲームは、得点差を状態にして後ろから minimax DP する

## 何に着目するか

二人が交互に最適な手を選ぶゲームでは、各マスから終点までの「高橋の得点 − 青木の得点」の最適値を決めれば十分です。手番はマスの座標和の偶奇で決まり、盤面は右・下にしか進まないので後ろから DP できます。

## 解法方針

`dp[i][j]` を、マス `(i,j)` から始めたときの最終得点差とします。ゴールは `dp[H-1][W-1]=0` です。

移動先の文字が `+` なら点数 `+1`、`-` なら `-1` とします。

|手番|選ぶ遷移|
|---|---|
|高橋（`(i+j)` 偶数）|`dp[next] + value[next]` の最大|
|青木（`(i+j)` 奇数）|`dp[next] - value[next]` の最小|

青木が `+` のマスへ進めば高橋との差は 1 減り、`-` のマスへ進めば差は 1 増るため、常に `-value[next]` を足す形です。

```text
高橋: 差を大きくする遷移を選ぶ
青木: 差を小さくする遷移を選ぶ
```

最終的に `dp[0][0]` が正なら Takahashi、負なら Aoki、0 なら Draw です。

## tips

### 実装

右下から左上へ走査します。各マスで右・下の盤面内の遷移だけを候補に入れます。高橋番の初期候補は十分小さい値、青木番は十分大きい値にします。

`value` を `+` なら `1`、`-` なら `-1` にしておくと式がそのまま書けます。

### よくある誤り

- 青木番でも遷移先の値を足す。得点差は高橋−青木なので引きます。
- 現在マスの文字を加点する。得点を得るのは**移動先**のマスです。
- 前から DP し、最適な両者の選択を混ぜる。後ろ向きに各状態のゲーム価値を確定します。

### 計算量

各マスで高々二遷移を調べるため、時間・メモリともに `O(HW)` です。
