# 059

## 問題リンク

[ABC285 F - Substring of Sorted String](https://atcoder.jp/contests/abc285/tasks/abc285_f)

## キーワード

文字列区間の複数条件は、独立な不変量に分けて管理する

## 何に着目するか

クエリ区間が条件を満たすには、二つの独立した条件があります。

1. 区間内の文字列自体が辞書順に非減少である。
2. 両端の文字の間にある文字種について、その文字の出現が区間外に残っていない。

一つ目は隣接二文字の逆転位置、二つ目は文字種ごとの出現数を管理すれば高速に判定できます。

## 解法方針

隣接位置 `i` に対し、`S[i] > S[i+1]` なら `bad[i]=1`、それ以外を `0` とします。区間 `[l,r]` が非減少である条件は、内部の `bad[l..r-1]` の和が 0 です。これは Fenwick Tree で一点更新・区間和取得できます。

次に各文字 `c` について、全体での出現数 `total[c]` と区間 `[l,r]` 内の出現数 `inside[c]` を求めます。

|文字 `c` の位置|必要なこと|
|---|---|
|`c = S[l]` または `c = S[r]`|区間外にあってもよい|
|`S[l] < c < S[r]`|`inside[c] == total[c]`|

なぜ中間文字だけ全出現が必要かというと、区間が非減少なら、文字列全体をソートしたときに中間文字は両端文字の間へ全て並ぶべきだからです。区間外に同じ中間文字が残ると、区間は連続部分文字列になれません。

```text
a ... c ... d  が区間の両端
      ↑
c が区間外にもある → a と d の間の連続部分にはできない
```

各文字の位置を Fenwick Tree で管理すれば `inside[c]` を区間和として得られます。

## tips

### 実装

文字更新で影響する隣接関係は `i-1` と `i` だけです。文字を書き換える前後でこの二つの `bad` を Fenwick Tree から削除・追加します。

文字種は 26 個なので、文字ごとに Fenwick Tree を一つ持っても `O(26N)` の範囲です。クエリでは端点の文字の間だけ走査します。

### よくある誤り

- 区間が非減少かだけを見る。中間文字が区間外にあるケースを見落とします。
- 両端文字についても全出現を要求する。端点文字は区間外にあっても連続部分として成立します。
- 更新で `bad[i-1]` だけ直す。変更文字は前後どちらの隣接関係にも影響します。

### 計算量

更新は `O(log N)`、判定は最大 26 文字の区間和取得で `O(26 log N)` です。メモリは `O(26N)` です。
