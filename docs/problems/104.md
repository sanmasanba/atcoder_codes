# 104

## 問題リンク

[ABC322 E - Product Development](https://atcoder.jp/contests/abc322/tasks/abc322_e)

## キーワード

各能力を目標値 P で丸めて持つ多次元最小コスト DP

## 何に着目するか

各能力は最終的に `P` 以上ならよく、`P+1` やそれ以上を区別しても答えに影響しません。能力値を各次元 `0..P` へ丸めれば状態数が有限になり、教材を選ぶ／選ばない 0/1 ナップサックになります。

## 解法方針

状態ベクトル `(x1,...,xK)` を「各能力の現在値（P で打ち切り）」とし、`dp[state]` をその状態を作る最小コストとします。

教材のコスト `C`、能力増分 `(A1,...,AK)` を選ぶと

```text
next_j = min(P, xj + Aj)
dp[next] = min(dp[next], dp[state] + C)
```

```text
能力 0..P-1 → 実値を持つ
能力 P      → 「P 以上」をまとめた状態
```

各教材は高々一度使うので、教材ごとに新しい `ndp` を `dp` から作るか、状態を逆順に走査します。全次元が `P` の状態の値が答えです。

## tips

### 実装

`K` 次元配列をそのまま使っても、基数 `P+1` で一つの整数 ID に符号化しても構いません。符号化する場合も、遷移時には各桁を取り出して個別に `min(P,...)` します。

到達不能の初期値は十分大きい `INF`、初期状態 `(0,...,0)` は 0 です。

### よくある誤り

- 能力値を P より大きく持ち続ける。状態数が不要に増えます。
- 教材を同じループ内で更新済み状態からも使い、複数回選んでしまう。0/1 選択です。
- 目標を各次元の和が P 以上と誤解する。全ての能力が個別に P 以上必要です。

### 計算量

状態数は `(P+1)^K` です。時間 `O(N(P+1)^K K)` 程度、メモリ `O((P+1)^K)` です。
