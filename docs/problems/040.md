# 040

## 問題リンク

[ABC283 F - Permutation Distance](https://atcoder.jp/contests/abc283/tasks/abc283_f)

## キーワード

絶対値を含む最小化は、大小関係で分けてデータ構造に載せる

## 何に着目するか

各 `i` について求めるのは

```text
min(j ≠ i) ( |i-j| + |P[i]-P[j]| )
```

です。`j` が `i` の左右どちらにあるか、`P[j]` が `P[i]` より上下どちらかを固定すると、絶対値が外れて一次式になります。残るのは「値が `P[i]` 以下／以上」という区間からの最小値取得です。

## 解法方針

まず `j<i`（左側の頂点）だけを考えます。`p=P[i]`, `q=P[j]` とします。

|条件|距離の式|左側に保存しておく値|
|---|---|---|
|`q < p`|`(i+p) - (j+q)`|`j+q` の最大値|
|`q > p`|`(i-p) + (q-j)`|`q-j` の最小値|

したがって、値 `q` を添字とするセグメント木を二本用意します。

- `q < p` の範囲には `-(j+q)` を保存し、区間最小値から最大値を得る
- `q > p` の範囲には `q-j` を保存する

左から右へ `i` を走査し、クエリしてから `P[i]` を二本の木へ追加します。これで `j<i` の候補が全て反映されます。

```text
左から右へ
  query(P[i])  : すでに見た j<i だけが対象
  update(P[i]) : 以後の i の候補として自分を登録
```

`j>i` の候補も必要なので、配列を反転して同じ処理をもう一度行います。反転後の添字を元へ戻して答えに `min` を取れば、右側の候補も含められます。

## tips

### 実装

`P` は順列なので値は `0..N-1`（または `1..N`）に座標圧縮済みです。値 `p` に対する範囲は、`q<p` が `[0,p)`、`q>p` が `(p,N)` です。空区間の取得値は十分大きい `INF` にします。

更新は同じ値に一度だけですが、汎用的には `set(value, min(old,new))` としておくと安全です。反転した走査用に、処理関数が「その走査で得た候補配列」を返す形にすると対称性を保てます。

### よくある誤り

- `q=p` のケースを入れる。順列なので `j≠i` なら値は必ず異なります。
- 左右だけ、あるいは値の大小だけで場合分けを終える。二つの絶対値なので 4 象限が必要です。
- 問い合わせ前に自分を更新する。`j=i` を候補に入れてしまいます。

### 計算量

二回の走査で各位置につき区間取得・更新を定数回行います。時間は `O(N log N)`、メモリは `O(N)` です。
