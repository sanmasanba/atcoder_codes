# 007

## 問題リンク

[ABC250 E - Prefix Equality](https://atcoder.jp/contests/abc250/tasks/abc250_e)

## キーワード

二つの集合を比較するときは、新しい要素に共通 ID を振って正規化する

## 何に着目するか

比べたいのは接頭辞の順番や重複ではなく、「そこまでに現れた値の集合」です。クエリごとに集合を作ると間に合いません。接頭辞を1つ伸ばしたときに、集合へ新しい値が加わったかだけを記録できないか考えます。

## 解法方針

まず配列 `A` で初めて現れた値に、`1, 2, 3, ...` とIDを振ります。`A` の接頭辞集合は常に `{1, 2, ..., k}` という形になります。

次に `B` を左から見ます。`B` の値が `A` に存在しないなら、その接頭辞は一致不能です。存在するなら、そこまでのIDの最大値を持ちます。未知値がなく、IDの最大値が `A` 側の接頭辞で得たID数と一致するときだけ、集合が一致します。

| 接頭辞 | 記録する値 | 意味 |
| --- | --- | --- |
| `A[0..i]` | 異なる値の個数 | 集合 `{1..k}` の `k` |
| `B[0..j]` | IDの最大値 | 必要な `A` の値がすべて現れたか |
| `B[0..j]` | 未知値の有無 | `A` にない値を含むか |

同じ値の再登場は集合を変えないため、ID数や最大IDを更新しません。

## tips

### 実装

辞書 `id_of` で `A` の値からIDを引けるようにします。`A` 側は新しい値の個数、`B` 側は最大IDと未知値フラグを前計算して、各クエリは定数時間で判定します。

### よくある誤り

- 接頭辞の長さが等しいと仮定する: 集合が同じでも重複数は違い得ます。
- `B` で同じIDが再登場したときに最大ID以外を増やす: 集合の大きさは増えません。
- 未知値をID 0として普通に比較する: 以後の接頭辞もすべて不一致になることを別途管理します。

### 計算量

辞書作成と前計算が `O(N)`、各クエリは `O(1)`、全体で `O(N + Q)` です。
