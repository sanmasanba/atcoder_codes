# 094

## 問題リンク

[ABC305 E - Art Gallery on Graph](https://atcoder.jp/contests/abc305/tasks/abc305_e)

## キーワード

届く強さを最大化する探索は、残り資源の大きい順に確定する

## 何に着目するか

警備員が頂点 `v` に体力 `h` でいると、距離 `d` の頂点まで `h-d≥0` の範囲を守れます。複数の警備員から届く場合は、残り体力が大きい経路だけを残せば十分です。

これは距離を最小化する Dijkstra と逆向きに、「残量を最大化する」優先度探索です。

## 解法方針

`best[v]` を頂点 `v` に届く残り体力の最大値とします。全警備員 `(v,h)` を最大ヒープへ入れ、最大残量から順に取り出します。

残量 `e` の頂点 `v` から隣 `to` へは `e-1` を渡れます。

```text
e-1 > best[to] なら
  best[to] = e-1
  heap へ追加
```

|残り体力|隣へ進めるか|
|---:|---|
|`e>0`|`e-1` を伝播できる|
|`e=0`|現在頂点は守るが、隣へは進めない|

最大ヒープから取り出した値が `best[v]` より小さければ古い候補なので捨てます。最後に `best[v]≥0` の頂点が守られる頂点です。

## tips

### 実装

Python の `heapq` は最小ヒープなので `-残量` を入れます。同じ頂点に複数警備員がいるときも、最大のものだけが有効ですが全て push して古い候補を捨てても構いません。

未到達を `-1` にすると、残量 0 との比較が自然です。

### よくある誤り

- 通常の BFS をする。初期体力が異なるので距離順だけでは不十分です。
- 残量 0 の頂点を未保護にする。そこまでは到達して守れます。
- より小さい残量で既訪問頂点を再探索する。最大値更新だけを採用します。

### 計算量

各辺が更新候補になる回数は定数回で、ヒープ操作を含めて `O((N+M+K) log N)`、メモリは `O(N+M)` です。
