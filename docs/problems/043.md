# 043

## 問題リンク

[ABC300 F - More Holidays](https://atcoder.jp/contests/abc300/tasks/abc300_f)

## キーワード

周期列の長区間は、完全周期と端の区間に分けて処理する

## 何に着目するか

対象文字列は `S` を `M` 回繰り返したものですが、長さは最大で非常に大きくなります。文字列全体を作る必要はありません。

区間を始める位置は周期 `N` ごとに同じです。開始位置を最初の一周期内に固定し、その後に取れる長さを「完全に含む周期」と「残りの一周期未満」に分けます。

## 解法方針

`S` に含まれる `x` の個数を `cx` とします。開始位置 `l (0≤l<N)` を固定したとき、`K` 個まで許される `x` で何周期分を丸ごと取れるかをまず求めます。

```text
q = K // cx     （cx > 0 の場合）
残り許容量 = K % cx
```

ただし文字列は高々 `M` 周期しかないので、完全周期数・最終長は `N*M-l` を超えないように切ります。残りについては `S+S` 上で、開始 `l` から `x` が残り許容量以下となる最長右端を二ポインタで求めます。

|区間の部分|長さ|`x` の個数|
|---|---:|---:|
|完全な周期|`q×N`|`q×cx`|
|最後の端（高々 1 周期）|二ポインタで決定|残り許容量以下|

二ポインタの右端は `l` を増やしても左へ戻りません。`S+S` を一度走査するだけで、全ての開始位置について端の最長長を求められます。

`cx=0` のときはどの区間にも `x` がないため、答えは全長 `N*M` です。

## tips

### 実装

開始位置ごとに答えを `min(N*M-l, 完全周期部分 + 端部分)` として更新します。`N*M` や答えは 64 bit 整数で持ちます。

二ポインタは `T=S+S` に対して `r` を単調に進め、`T[l:r]` の `x` 数が残り許容量以下になる最大の `r` を維持します。端部分の長さは高々 `N` に制限します。

### よくある誤り

- `S` を `M` 回連結する。`M` が大きく、メモリも時間も足りません。
- 完全周期の数をそのまま使い、末尾を越える。開始位置によって残り総長が変わります。
- `cx=0` で `K//cx` を計算する。先に特別扱いします。

### 計算量

一周期の開始位置 `N` 個を扱い、二ポインタは `S+S` を高々一度ずつ進むため、時間は `O(N)`、メモリは `O(N)` です。
