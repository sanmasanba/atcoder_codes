# 031

## 問題リンク

[ABC243 E - Edge Deletion](https://atcoder.jp/contests/abc243/tasks/abc243_e)

## キーワード

辺が不要か判定するなら、全点対最短距離で代替経路を調べる

## 何に着目するか

ある辺 `(a,b,c)` が必要かどうかは、端点 `a,b` の間にその辺を使わない同等以下の経路があるかで決まります。頂点数が小さいため、全点対最短距離を一度求めれば、各辺を局所的に判定できます。

## 解法方針

まず Warshall-Floyd 法で `dist[i][j]` を全点対最短距離にします。辺 `(a,b,c)` が不要になるのは次の二通りです。

|条件|意味|判定|
|---|---|---|
|もっと短い経路がある|この辺を通る最短路は存在しない|`dist[a][b] < c`|
|同じ長さの別経路がある|この辺を使わない最短路が存在する|ある `k` で `dist[a][k]+dist[k][b] == c`|

後者では `k` を `a,b` 以外から選びます。辺の重みは正なので、`a` から `k` への最短経路または `k` から `b` への最短経路がこの辺を使っているなら、合計は `c` より大きくなります。従って等式が成り立てば、実際にその辺を使わない代替経路があります。

```text
a --- c --- b      辺そのもの
 \           /
  dist[a][k] + dist[k][b] = c
        k

→ c と同じ長さの迂回路があるので、辺 (a,b) は消せる
```

複数辺がある場合も、`dist[a][b]` は最短の一本だけを反映します。長いほうは最初の条件で不要と判定されます。

## tips

### 実装

`dist` は `INF` で初期化し、対角だけ `0` にします。入力辺を読む際は `dist[a][b] = dist[b][a] = min(dist[a][b], c)` とし、判定のために各入力辺 `(a,b,c)` は別配列へすべて保存します。

Warshall-Floyd の更新は `dist[i][k] + dist[k][j]` が `INF` でないことを確認してから行うと安全です。

### よくある誤り

- `dist[a][b] == c` なら必要だと決める。同じ長さの別経路があれば不要です。
- `k=a` や `k=b` を代替経路として数える。これは辺そのものを使う自明な式になってしまいます。
- 同じ端点の複数辺を一つしか保存しない。答えは入力された辺ごとに数えます。

### 計算量

Warshall-Floyd が `O(N^3)`、各辺の判定が `O(NM)` です。この問題の制約では `O(N^3 + NM)` で十分です。メモリは `O(N^2 + M)` です。
