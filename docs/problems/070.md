# 070

## 問題リンク

[ABC234 F - Reordering](https://atcoder.jp/contests/abc234/tasks/abc234_f)

## キーワード

重複を含む列の数え上げは、種類ごとの挿入位置の組合せで DP する

## 何に着目するか

同じ文字は区別しません。文字種を `a,b,c,...` の順に処理し、これまでに作った文字列へ新しい文字を何個挿入するかを選べば、重複なく全ての文字列を数えられます。

新しい文字を `j` 個加えるとき、その挿入位置の選び方は組合せです。

## 解法方針

`dp[len]` を「処理済み文字種だけを使って長さ `len` の文字列を作る方法数」とします。新しい文字種の出現数を `c` とし、既存長 `len` の文字列へ `j (0≤j≤c)` 個を加えるとします。

最終長は `len+j` で、新文字を置く位置の選び方は

```text
C(len+j, j)
```

です。

|遷移|意味|
|---|---|
|`dp[len] → dp[len+j]`|新文字を `j` 個使う|
|係数 `C(len+j,j)`|全体 `len+j` 個から新文字の位置を選ぶ|

全 26 文字種を処理した後、空文字列を除く `Σ dp[len] (len≥1)` が答えです。

```text
"ab" に c を 2 個追加
長さ4のうち c の位置を選ぶ → C(4,2) 通り
```

## tips

### 実装

最大長 `N=|S|` までの階乗・逆階乗を前計算し、`C(n,r)=fact[n]*ifact[r]*ifact[n-r]` とします。文字種ごとに新しい配列 `ndp` を作ると、同じ文字種を過剰に使うことがありません。

`j=0` の遷移も必要です。その文字を使わない文字列を保ちます。

### よくある誤り

- 同じ文字を順列として区別する。`c` 個の新文字の順序は一通りです。
- `C(len,j)` を掛ける。挿入後の全長 `len+j` の位置から選びます。
- 空文字列を答えに含める。問題は非空文字列だけです。

### 計算量

文字種ごとに長さと使用数を走査するため、時間 `O(26N^2)`、メモリ `O(N)` です。
