# 022

## 問題リンク

[ABC265 E - Warp](https://atcoder.jp/contests/abc265/tasks/abc265_e)

## キーワード

少ない操作回数の格子移動は、操作回数と移動回数の組で状態を表す

## 何に着目するか

盤面は無限ですが、各手で選べる移動は 3 種類、手数は高々 `N` です。同じ移動を何回選んだかが分かれば、現在位置は一意に決まります。座標そのものを広い範囲で DP する必要はありません。

## 解法方針

3 種類の移動ベクトルを `(a1,b1)`, `(a2,b2)`, `(a3,b3)` とします。`t` 手目までに、1 番目を `i` 回、2 番目を `j` 回使った状態を考えます。3 番目の回数は `t-i-j` 回に決まります。

このとき位置は次の式で復元できます。

```text
x = i*a1 + j*a2 + (t-i-j)*a3
y = i*b1 + j*b2 + (t-i-j)*b3
```

したがって `dp[i][j][k]` を「各移動を `i, j, k` 回使って障害物を踏まずに到達する方法数」として持てば、各状態から 3 方向へ遷移するだけです。遷移先の座標が障害物集合に入っているときだけ、その遷移を捨てます。

|状態|表すもの|次の状態|
|---|---|---|
|`(i,j,k)`|3 種類をそれぞれ使った回数|`(i+1,j,k)`|
|||`(i,j+1,k)`|
|||`(i,j,k+1)`|

状態の座標が同じになることがあっても問題ありません。問われているのは移動列の数なので、異なる移動列は DP 値として加算します。

## tips

### 実装

`dp` は `N+1` の 3 次元配列にして、`i+j+k < N` の状態だけ処理します。遷移先の座標はその場で式から計算し、障害物は `(x, y)` の組を `set` に入れて `O(1)` 平均で判定します。

答えは `i+j+k=N` を満たすすべての `dp[i][j][k]` の和です。各加算のたびに `998244353` で剰余を取ります。

### よくある誤り

- 到達した時点ではなく、障害物への**遷移**を止める。障害物のマスには一度も乗れません。
- 3 種類の使用回数の和が `N` の状態を一つだけだと思う。最終位置も移動列も複数あります。
- 座標の範囲を配列にする。移動量が大きいため、回数を添字にするほうが安全です。

### 計算量

`i+j+k≤N` の状態数は `O(N^3)` です。時間・メモリともに `O(N^3)` です。
