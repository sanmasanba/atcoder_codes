# 117

## 問題リンク

[ABC335 E - Non-Decreasing Colorful Path](https://atcoder.jp/contests/abc335/tasks/abc335_e)

## キーワード

同値辺を縮約して値が厳密増加する DAG 上でスコア DP をする

## 何に着目するか

同じ値の頂点間はスコアを増やさず自由に行き来できます。先に同値の辺で連結な頂点を一成分へ縮約すると、成分間の辺は値が異なるため、小さい値から大きい値へ向けた DAG として扱えます。

## 解法方針

値が等しい両端を持つ辺で Union-Find を併合します。成分 `c` の値は所属頂点の値です。異なる値の辺は、小さい成分から大きい成分への有向辺とみなします。

`dp[c]` を始点から成分 `c` まで到達したときの最大スコアとします。始点成分のみ `1`、未到達は 0 です。値順に成分を処理し、辺 `c→to` で

```text
dp[to] = max(dp[to], dp[c] + 1)
```

と更新します。

```text
同値成分: 0 点で自由移動
値が増える辺: スコア +1
```

終点成分の `dp` が答えです。到達できなければ 0 のままです。

## tips

### 実装

全頂点を値昇順にソートして処理し、各頂点から大きい値の隣へだけ緩和しても、成分ごとに隣接リストを作っても構いません。同値の辺を先に併合し終えることが重要です。

始点と終点が同じ成分なら、始点のスコア 1 がそのまま答えです。

### よくある誤り

- 同値辺を `+1` の遷移にする。同じ値ではスコアは増えません。
- 同値成分を縮約せず、処理順で循環させる。値が等しい辺により DAG でなくなります。
- 未到達の `dp=0` から `+1` 更新する。`dp[c]>0` の成分だけ緩和します。

### 計算量

Union-Find と辺走査、値ソートで `O((N+M)α(N)+N log N)`、メモリ `O(N+M)` です。
