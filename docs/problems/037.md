# 037

## 問題リンク

[ABC230 E - Fraction Floor Sum](https://atcoder.jp/contests/abc230/tasks/abc230_e)

## キーワード

整数除算の商が等しい範囲をまとめて、平方根個の区間に圧縮する

## 何に着目するか

求める値は `Σ floor(N/i)` です。`i` を 1 ずつ増やすと最大 `N` 回必要に見えますが、整数除算の商 `floor(N/i)` は長い区間で同じ値を取ります。

左端 `l` における商を `q=N//l` とすると、同じ商が続く最後の添字は直接求められます。

## 解法方針

`floor(N/i)=q` となる最大の `i` は `r=N//q` です。従って区間 `[l,r]` の全項は `q` で、寄与を一度に加算できます。

```text
l = 4, N//l = 3
N//i = 3 となる最大 i は N//3

[4, N//3] の寄与 = 3 × (N//3 - 4 + 1)
```

処理後は `l=r+1` に進めます。

|左端 `l`|商 `q=N//l`|右端 `r=N//q`|加える値|
|---:|---:|---:|---:|
|`l`|`q`|`r`|`q×(r-l+1)`|

商の異なる値は高々およそ `2√N` 種類です。小さい商の部分は区間が長く、大きい商の部分はそもそも種類が少ないためです。

## tips

### 実装

`l=1` から `l<=N` の間ループします。`q=N//l`, `r=N//q`、`ans += q*(r-l+1)`、`l=r+1` の 4 行が中心です。

答えは `N log N` 程度まで大きくなるため、C++ では `long long` を使います。

### よくある誤り

- `r=N//q-1` とする。右端は `N//q` を含みます。
- `l` を `r` へ進める。同じ区間を再び処理するため `r+1` です。
- 浮動小数点で境界を求める。整数除算だけで正確に求められます。

### 計算量

商の値が変わる区間だけを処理するため、時間は `O(√N)`、追加メモリは `O(1)` です。
