# 088

## 問題リンク

[ABC296 E - Transition Game](https://atcoder.jp/contests/abc296/tasks/abc296_e)

## キーワード

機能グラフの閉路を取り出すなら、入次数 0 を連鎖的に削る

## 何に着目するか

各頂点の行き先が一つの機能グラフでは、どの頂点から進んでも最終的に閉路へ入ります。質問の条件を満たすのは、開始頂点へ戻って来られる、すなわち閉路上にある頂点だけです。

木の葉を剥がすのと同様に、入次数 0 の頂点は閉路上にあり得ません。これを連鎖的に削除します。

## 解法方針

全頂点の入次数を数え、入次数 0 の頂点をキューへ入れます。キューから `v` を取り出して「削除済み」にし、行き先 `to=A[v]` の入次数を一つ減らします。`to` が 0 になればキューへ入れます。

```text
入次数 0 の頂点 → 閉路へ入るだけで戻れない → 削除
削除により新たな入次数 0 が生まれる → 続けて削除
最後に残る頂点 → 全て閉路上
```

|頂点の状態|意味|
|---|---|
|削除された|閉路へ流れ込む木部分|
|削除されず残った|閉路上|

キュー処理の後に削除されず残った頂点数が答えです。

## tips

### 実装

`indeg` と `removed`（または最終 `indeg>0`）を持ちます。各頂点は高々一度キューへ入り、出辺も一度だけ処理されます。

行き先配列は 0-indexed に直しておくと添字のずれを避けられます。

### よくある誤り

- 入次数 1 の頂点も最初に削除する。閉路上の頂点は入次数 1 以上なので削除してはいけません。
- キューから取り出した後、行き先の入次数を減らさない。連鎖的な葉剥がしが起きません。
- 残った頂点を「連結成分の代表」と解釈する。残るのは閉路の全頂点です。

### 計算量

各頂点・出辺を一度処理するため、時間・メモリともに `O(N)` です。
