# 096

## 問題リンク

[ABC312 F - Cans and Openers](https://atcoder.jp/contests/abc312/tasks/abc312_f)

## キーワード

補助資源に使う個数を固定して、残り枠の最適選択を考える

## 何に着目するか

缶切り自体は価値 0 ですが、開封缶を開ける能力を増やします。缶切りを何個選ぶかを固定すれば、残りの選択枠で取れる通常缶・開封済み缶の価値上位を選ぶだけです。

## 解法方針

通常缶（type 0）と開封缶（type 1）の価値を降順、缶切り（type 2）の開封可能数を降順に並べます。缶切りを一個ずつ追加し、累積開封可能数 `cap` を増やします。

その時点で使える開封缶は上位 `min(cap, type1の個数)` 個です。通常缶と使える開封缶を合わせた中から、残り枠 `M-缶切り数` 個の価値上位を取ります。

```text
缶切り k 個
  → 開封可能数 cap
  → 残り M-k 枠を type0 と先頭 cap 個の type1 から選ぶ
```

価値候補を大きい順に管理するヒープ／二集合を使い、缶切り数を増やすごとに新たに使える type 1 を追加します。各段階の選択価値和の最大を答えにします。

## tips

### 実装

缶切りを増やす前後で残り枠が一つ減る点に注意します。選択中集合を「価値上位の残り枠個」として保ち、追加候補や枠減少に合わせて最小要素を抜きます。缶切りを一つも使わないケースも評価します。

### よくある誤り

- 開封缶を価値順に無制限に選ぶ。type 2 の累積能力までです。
- 缶切りの個数を選択枠 M に含めない。缶切りも一品です。
- type 1 だけを優先する。通常缶との全候補から上位を選びます。

### 計算量

ソートとヒープ更新を含めて `O(N log N)`、メモリ `O(N)` です。
