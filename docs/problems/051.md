# 051

## 問題リンク

[ABC271 E - Subsequence Path](https://atcoder.jp/contests/abc271/tasks/abc271_e)

## キーワード

使える辺の順序が固定なら、その順に一回ずつ緩和する DP にする

## 何に着目するか

使える辺の番号列が与えられ、その順番を保った部分列だけを選べます。これは通常のグラフ最短路ではなく、「辺を一つ読むたびに使う／使わないを選ぶ」一次元 DP です。

辺を読む順番が決まっているため、その時点までに作れる最短距離だけを持てば十分です。

## 解法方針

`dist[v]` を「ここまでに読んだ指定辺の部分列で頂点 `v` へ行く最短距離」とします。初期状態は `dist[0]=0`、それ以外は `INF` です。

指定された辺番号 `e` に対応する辺を `(a,b,c)` とすると、辺を使う選択は次です。

```text
dist[b] = min(dist[b], dist[a] + c)
```

|選択|`dist` の意味|
|---|---|
|辺を使わない|配列を変えない|
|辺 `a→b` を使う|`dist[b]` を `dist[a]+c` で緩和|

辺は今読んでいる一回しか使えないため、同じ `dist` 配列をその場更新して構いません。辺が一方向なので、更新後の `dist[b]` が同じ辺で `a` へ戻って使われることもありません。

最後に `dist[N-1]` が `INF` なら到達不能で `-1`、そうでなければその値が答えです。

## tips

### 実装

辺を `(from,to,cost)` として 0-indexed で保存し、指定列の各番号を読むたびに一回だけ緩和します。`dist[a]==INF` なら加算せずに飛ばします。

距離の最大値に備え、C++ では `long long` と十分大きい `INF` を使います。

### よくある誤り

- 通常の Dijkstra / Bellman-Ford を行う。使える順序の制約を無視してしまいます。
- 辺列の各位置で `new_dist` を別に作る必要があると考える。この問題は辺を一度しか使わない有向更新なので、その場更新で「使わない」遷移も保たれます。
- `dist[a]` が未到達のまま `INF+c` を計算する。

### 計算量

指定辺列の長さを `K` とすると、一回につき定数時間で更新するため `O(K)`、メモリは `O(N+M)` です。
