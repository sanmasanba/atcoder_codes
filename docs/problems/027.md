# 027

## 問題リンク

[ABC223 F - Parenthesis Checking](https://atcoder.jp/contests/abc223/tasks/abc223_f)

## キーワード

括弧列の区間判定は、和と最小累積和に分解する

## 何に着目するか

`(` を `+1`、`)` を `-1` とみなします。区間 `[l,r]` が正しい括弧列である条件は二つだけです。

1. 区間全体の和が `0`
2. 左から読んだ累積和が一度も負にならない

交換クエリは高々 2 箇所の値を変える操作です。区間和と区間内の最小累積和をセグメント木で持ちます。

## 解法方針

各区間について、次の組を保持します。

```text
(sum, min_prefix)
sum        : 区間全体の和
min_prefix : 区間の左端を 0 とした累積和の最小値
```

左区間 `L` と右区間 `R` をつなぐときの結合は次です。

|値|結合結果|
|---|---|
|`sum`|`L.sum + R.sum`|
|`min_prefix`|`min(L.min_prefix, L.sum + R.min_prefix)`|

右区間の累積和は、左区間を読み終えた時点の `L.sum` だけ上へずれるため、`R.min_prefix` に `L.sum` を足すのが要点です。

たとえば `L="()"` は `(0,0)`、`R=")("` は `(0,-1)` なので、連結した `"())("` は `(0,-1)` となり、途中で負になることが分かります。

クエリ 1 は、`S[l]` と `S[r]` が異なるときだけ 2 点更新します。クエリ 2 は区間の組を取得し、`sum == 0` かつ `min_prefix >= 0` なら `Yes` です。

## tips

### 実装

葉は `(` なら `(1, 0)`、`)` なら `(-1, -1)` です。空区間の単位元は `(0, 0)` にします。Python では `atcoder.segtree.SegTree` の演算 `op` として上の結合を定義できます。

入力の添字は 1-indexed なので、文字列・セグメント木へ渡す前に `l-1`, `r-1` へ直します。`prod(l, r)` の右端は半開区間です。

### よくある誤り

- 総和だけで判定する。`)(` の和も 0 ですが正しい括弧列ではありません。
- `min(R.min_prefix, L.sum + R.min_prefix)` としてしまう。右区間の最小値は必ず左区間の和だけ平行移動します。
- 同じ括弧を交換しても更新する。値が変わらないので、そのままにして構いません。

### 計算量

構築は `O(N)`、2 点更新と区間取得はそれぞれ `O(log N)` です。メモリは `O(N)` です。
