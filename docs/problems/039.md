# 039

## 問題リンク

[ABC280 F - Pay or Receive](https://atcoder.jp/contests/abc280/tasks/abc280_f)

## キーワード

差分制約を併合するときは、重み付き Union-Find で矛盾も判定する

## 何に着目するか

辺は `value[b] - value[a] = c` という差の制約です。同じ連結成分なら、ある基準頂点からの差を足し合わせて二頂点の差を求められます。ただし閉路の制約が食い違う成分では、値を一意に決められません。

「つながっているか」と「根までの差」を同時に管理する重み付き Union-Find を使います。

## 解法方針

各頂点 `v` について、`weight[v] = value[v] - value[root(v)]` を保持します。経路圧縮時に親の重みを足せば、根までの差を更新できます。

同じ根を持つ `a,b` なら、二頂点の差は

```text
value[b] - value[a] = weight[b] - weight[a]
```

です。新しい制約 `(a,b,c)` を加えるときの処理は次の通りです。

|根の関係|処理|
|---|---|
|異なる成分|`value[b]-value[a]=c` を満たすように根を併合する|
|同じ成分で `weight[b]-weight[a]==c`|矛盾なし|
|同じ成分で異なる|その成分を「矛盾あり」と印付けする|

併合時、`ra=root(a)`, `rb=root(b)` として `ra` を `rb` の子にするなら、親辺に持たせる差は

```text
value[ra] - value[rb] = weight[b] - weight[a] - c
```

です。符号を暗記せず、この式を一度書いてから実装します。併合する二成分の `bad` フラグは OR を取ります。

クエリ `(x,y)` は次の三分岐です。

```text
別成分                 → nan
同じ成分かつ bad       → inf
同じ成分かつ矛盾なし   → weight[y] - weight[x]
```

## tips

### 実装

`find(v)` は根を返すだけでなく、再帰から戻るときに `weight[v] += weight[parent[v]]` をして差を更新します。`find(a)`, `find(b)` を呼んだ後なら、`weight[a]`, `weight[b]` はそれぞれ根までの差です。

Union by size/rank をする場合、根の向きを反転したときは新しい親辺の重みの符号も反転します。

### よくある誤り

- 矛盾する辺を見たら併合を取り消す。既に同じ成分なので連結性は変わりません。必要なのは成分の `bad` 化です。
- `weight[x]-weight[y]` を出力する。制約の向きを `value[y]-value[x]` と統一します。
- 経路圧縮後に `weight` を更新しない。根までの差が古い親を基準にしたままになります。

### 計算量

各辺・クエリは Union-Find 操作数回で、ならし `O(α(N))` です。メモリは `O(N)` です。
