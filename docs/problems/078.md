# 078

## 問題リンク

[ABC259 F - Select Edges](https://atcoder.jp/contests/abc259/tasks/abc259_f)

## キーワード

次数制約つき木 DP は、親辺の有無で残り枠を分けて利得上位を選ぶ

## 何に着目するか

頂点 `v` で選べる辺数は `D[v]` 本までです。子辺を選ぶかどうかは子部分木の最適値に影響しますが、親辺を既に選んだかどうかだけが `v` 側で必要な情報です。

子辺を選ぶ追加の得は独立に計算できるため、次数枠に入る正の利得だけを大きい順に選べばよいです。

## 解法方針

根付き木として、各頂点 `v` に二値を定義します。

|値|意味|
|---|---|
|`free[v]`|親辺を選ばないときの `v` 部分木の最大値|
|`used[v]`|親辺を選んだときの最大値（`D[v]=0` なら不可）|

子 `to` を見ると、辺 `(v,to)` を選ばない基準は `free[to]`、選ぶと `used[to]+w` になります。よって子辺を選ぶ純増分は

```text
gain[to] = used[to] + w(v,to) - free[to]
```

まず全ての `free[to]` を足し、`gain` の正のものを降順に並べます。

```text
free[v] = 基準和 + 上位 D[v] 個の正の gain
used[v] = 基準和 + 上位 D[v]-1 個の正の gain
```

親辺を選んだ `used[v]` では、`v` の次数枠を親辺が一つ使うため、子へ割ける枠が一つ少なくなります。

```text
親辺なし:  子辺を最大 D[v] 本
親辺あり:  子辺を最大 D[v]-1 本
```

## tips

### 実装

DFS を子から親へ戻る順で行います。`D[v]=0` の `used[v]` は `-INF` とし、親がその辺を選ばないようにします。

各頂点で子数ぶんの `gain` をソートします。合計値は 64 bit 整数で持ちます。

### よくある誤り

- 負の gain まで次数枠が残っているからと選ぶ。辺を選ばないほうがよいので正のものだけです。
- `used[v]` でも `D[v]` 本選べるとする。親辺が一枠を消費しています。
- 子辺選択時に `free[to]` を使う。子側でも親辺を選んだ状態 `used[to]` です。

### 計算量

各頂点で子の gain をソートするため、合計 `O(N log N)` 程度、メモリ `O(N)` です。
