# 076

## 問題リンク

[ABC251 E - Takahashi and Animals](https://atcoder.jp/contests/abc251/tasks/abc251_e)

## キーワード

環状の選択 DP は、一頂点を選ぶ・選ばないで固定して直線にする

## 何に着目するか

各動物を起こすには、自分または隣の動物へ餌を与える必要があります。これは環グラフの各辺を少なくとも片端で覆う「最小重み頂点被覆」です。

環のままでは先頭と末尾がつながりますが、先頭を選ぶか選ばないかを固定すれば直線になります。

## 解法方針

位置 `i` までを処理し、`dp[i][0/1]` を「`i` を選ばない／選ぶ最小コスト」とします。直線上では、`i` を選ばないなら辺 `(i-1,i)` を覆うため `i-1` は必ず選ばれていなければなりません。

|現在 `i`|遷移|
|---|---|
|選ばない|`dp[i][0] = dp[i-1][1]`|
|選ぶ|`dp[i][1] = min(dp[i-1][0], dp[i-1][1]) + A[i]`|

先頭について二ケースを計算します。

```text
case 1: 0 を選ぶ    → 初期 dp[0] = [INF, A[0]]
case 2: 0 を選ばない → 初期 dp[0] = [0, INF]
```

最後に、case 2 では環の辺 `(N-1,0)` を覆うため末尾 `N-1` を選んだ状態だけを許します。case 1 では末尾のどちらでも構いません。二ケースの最小が答えです。

## tips

### 実装

各ケースで前の二値だけを持てばよく、配列全体は不要です。`INF` はコスト総和より十分大きく取ります。

`N=2` でも式はそのまま適用できますが、初期・終端の許可状態を紙に書いて確認します。

### よくある誤り

- `i` を選ばない遷移で `min(dp[i-1])` を使う。隣接辺を覆うため前は必ず選択済みです。
- 先頭の状態を一つだけで始める。環の最後との辺が処理できません。
- case 2 の末尾未選択を許す。先頭も末尾も選ばれず、環の辺が未被覆です。

### 計算量

二ケースを各 `N` 頂点で処理するため、時間 `O(N)`、追加メモリ `O(1)` です。
