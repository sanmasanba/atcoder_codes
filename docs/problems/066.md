# 066

## 問題リンク

[ABC224 E - Integers on Grid](https://atcoder.jp/contests/abc224/tasks/abc224_e)

## キーワード

大小関係だけの DP は、値順に処理し同値を同時に更新する

## 何に着目するか

あるセルから遷移できるのは、同じ行または列にあり値が**より大きい**セルだけです。値が大きい順なら、遷移先の DP 値はすでに分かっています。

ただし値が同じセル同士は遷移できません。同値セルを一つ処理するたびに行・列の最大値を更新すると、存在しない遷移を許してしまいます。

## 解法方針

`row[r]` を「すでに処理した、行 `r` のより大きい値のセルにおける DP 最大値」、`col[c]` も同様にします。値 `a` の同値グループ内のセル `(r,c)` は

```text
dp[r,c] = max(row[r], col[c])
```

で計算できます。

```text
値 a のグループ:
  1. 全セルの dp を row / col の現状から計算
  2. グループ全体を計算し終えてから row / col を更新
```

|処理済みの値|行・列最大へ反映するか|
|---|---|
|現在値より大きい|する|
|現在値と等しい|グループ終了までしない|
|現在値より小さい|まだ処理しない|

グループの計算後、各セルについて `row[r]=max(row[r],dp+1)`、`col[c]=max(col[c],dp+1)` と更新します。`+1` はそのセルを一手として使うためです。

## tips

### 実装

与えられたセルだけを `(A,r,c,入力番号)` としてソートします。行・列の配列は大きさ `H`,`W` で 0 初期化し、答えを入力順へ保存します。

同じ `A` の範囲をポインタで取り出し、まず答え配列へ全て書き込んでから二周目で `row`,`col` を更新します。

### よくある誤り

- 同値グループ内で即時更新する。同じ値へ移れることになり誤りです。
- `dp` を `row+1` と `col+1` の最大にする。ここでの `row`,`col` は既に「遷移回数」を表す値なので、更新側でだけ `+1` をします。
- 空の行・列を `-1` にし、初期ケースをずらす。0 初期化で大きい値がないときの答え 0 が自然に出ます。

### 計算量

セル数を `N` とすると、ソートが `O(N log N)`、グループ処理は `O(N)`、メモリは `O(N+H+W)` です。
