# 024

## 問題リンク

[ABC255 E - Lucky Numbers](https://atcoder.jp/contests/abc255/tasks/abc255_e)

## キーワード

交互符号の漸化式は、初項の一次式として展開する

## 何に着目するか

`A[i] + A[i+1] = S[i]` という条件だけでは数列は一意に決まりません。しかし `A[1]` を決めれば、以後はすべて復元できます。自由度が一つしかないことを利用して、その初項をどれにすべきか数え上げます。

## 解法方針

`A[1] = x` とおきます。式を順に使うと、各項は `x` の符号付き一次式になります。

```text
A[1] =  x
A[2] =  S[1] - x
A[3] =  S[2] - S[1] + x
A[4] =  S[3] - S[2] + S[1] - x
```

そこで `A[i] = sign[i] * x + offset[i]` と表します。`sign` は `+1, -1, +1, ...` と交互になり、`offset` は `S` を交互に足し引きして更新できます。

目標値 `X[j]` を `A[i]` に含めるには、必要な `x` は一意です。

|`sign[i]`|条件|`x`|
|---:|---|---|
|`+1`|`x + offset[i] = X[j]`|`X[j] - offset[i]`|
|`-1`|`-x + offset[i] = X[j]`|`offset[i] - X[j]`|

全ての `(i,j)` についてこの候補 `x` を辞書で数えます。同じ `x` が多く現れるほど、その初項で数列を作ったときに `X` の要素と一致する項が多くなります。辞書の最大頻度が答えです。

## tips

### 実装

先頭から `offset=0`, `sign=1` として各 `A[i]` を処理し、その後に `A[i+1] = S[i] - A[i]` に対応する更新をします。あるいは、`T[0]=0`, `T[i+1]=S[i]-T[i]` を作り、`A[i]` の符号を添えて扱っても構いません。

値は負になり、`S` の累積で大きくなります。C++ なら `long long`、Python なら `int` を使います。

### よくある誤り

- `A[i]` の `x` の符号を常に正にする。隣接和から復元するたびに符号が反転します。
- `X[j]` ごとに数列全体を復元する。初項候補だけを数えると `O(NM)` で済みます。
- 一致する「値」の種類数を数える。この問題では一致する添字の個数を最大化します。

### 計算量

候補を `N×M` 個作るため、時間は `O(NM)`、辞書の要素数も最悪 `O(NM)` です。
