# 055

## 問題リンク

[ABC278 E - Grid Filling](https://atcoder.jp/contests/abc278/tasks/abc278_e)

## キーワード

各要素が有効な配置範囲は、長方形加算にして二次元 imos で重ねる

## 何に着目するか

切り抜きの外に色が残らないのは、その色の全出現マスが切り抜き内へ収まるときです。色ごとの全出現を一つずつ追う代わりに、最小／最大の行・列からなる外接長方形だけを見れば判定できます。

## 解法方針

色 `c` の出現範囲を `[rmin,rmax]×[cmin,cmax]` とします。高さ `h`、幅 `w` の切り抜き左上を `(r,c)` としたとき、色が完全に消える条件は

```text
r ≤ rmin,  r+h-1 ≥ rmax
c ≤ cmin,  c+w-1 ≥ cmax
```

です。したがって `(r,c)` が入る範囲は

```text
r ∈ [rmax-h+1, rmin]
c ∈ [cmax-w+1, cmin]
```

（有効な切り抜き左上範囲へ切り詰める）。この長方形に色ごとに二次元 imos で `+1` します。

```text
imos 値 = その切り抜きで完全に消える色数
答え     = 全色数 - imos 値
```

## tips

### 実装

各色について min/max 行列を前計算します。出現しない色は無視します。更新長方形が空なら加算しません。

二次元 imos の差分配列を二方向に累積和し、全ての切り抜き左上で答えを出します。

### よくある誤り

- 色の出現数だけを使う。必要なのは全出現を覆う外接範囲です。
- 左上の範囲を盤面全体で考え、切り抜き可能範囲 `0..H-h`, `0..W-w` へ切り詰めない。
- imos 値をそのまま答えにする。求めるのは外に残る色数なので全色数から引きます。

### 計算量

前処理と二次元 imos が `O(HW+N)` 程度、メモリ `O(HW+N)` です。
