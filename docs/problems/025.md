# 025

## 問題リンク

[ABC202 E - Count Descendants](https://atcoder.jp/contests/abc202/tasks/abc202_e)

## キーワード

部分木への問い合わせは、DFS 順の連続区間に平坦化する

## 何に着目するか

クエリは「頂点 `u` の部分木にあって、深さが `d` の頂点はいくつか」です。木を毎回たどると間に合いません。DFS で頂点に入った順番を使うと、部分木は 1 本の配列の連続区間になります。

深さまで固定されたクエリなので、「深さごとに DFS 順を並べた配列」を用意すれば、区間内にある要素数へ帰着できます。

## 解法方針

根から DFS し、頂点 `v` に入った時刻を `tin[v]`、`v` の部分木をすべて探索し終えた時刻を `tout[v]` とします。半開区間で考えると、`v` の部分木に属する頂点の `tin` はちょうど `[tin[v], tout[v])` に入ります。

同時に、深さ `d` の頂点の `tin` を DFS で訪れた順に `depth_nodes[d]` へ追加します。DFS 順に追加しているので、各配列は昇順です。

```text
DFS 順:       0   1   2   3   4   5
頂点:         r   a   c   d   b   e

a の部分木:       [1-------------4)
深さ 2 の tin:        [2, 3, 5]
                 lower_bound(1) = 0
                 lower_bound(4) = 2  → 2 個
```

クエリ `(u,d)` の答えは、`depth_nodes[d]` に対して次で求められます。

```text
lower_bound(tout[u]) - lower_bound(tin[u])
```

## tips

### 実装

再帰 DFS は Python では深くなりやすいため、`sys.setrecursionlimit` を十分大きくするか、明示スタックで入時・出時を処理します。`tout` は「最後に訪れた頂点の時刻」ではなく、部分木の次の時刻（右端）にします。

存在しない深さへのクエリは `0` です。深さごとの配列を `defaultdict(list)` にするとこの場合も扱いやすくなります。

### よくある誤り

- `tout` を閉区間の右端として作り、`bisect_left` と混ぜる。`[tin, tout)` に統一すると境界を間違えにくくなります。
- 深さごとの頂点番号を保存するだけで、DFS 順を保存しない。二分探索する対象は `tin` です。
- `u` 自身を部分木から除く。問題の「子孫」は `u` 自身を含む定義です。

### 計算量

DFS と配列構築は `O(N)`、各クエリは二分探索 2 回で `O(log N)` です。全体で `O(N + Q log N)`、メモリは `O(N)` です。
