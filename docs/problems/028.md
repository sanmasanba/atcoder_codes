# 028

## 問題リンク

[ABC228 D - Linear Probing](https://atcoder.jp/contests/abc228/tasks/abc228_d)

## キーワード

埋まった位置を次の未使用位置へ飛ばすなら、Union-Find で後継を管理する

## 何に着目するか

挿入先は `h=x mod 2^20` から右へ進んで最初に空いているマスです。衝突のたびに 1 マスずつ調べると、同じ埋まった区間を何度も通る可能性があります。

一度埋まったマスは、以後空きに戻りません。この単調性があるため、「この位置以上で最初に空いている場所」を Union-Find の代表として持てます。

## 解法方針

配列長を `M=2^20` とします。`find(i)` を「`i` 以上で最初に空いている添字」と定義します。`i` 自身が空いていれば `find(i)=i`、`i` が埋まったら親を次へつなぎます。

マス `p` を埋めた直後に、次のようにします。

```text
parent[p] = find(p + 1)
```

これで、次回 `find(p)` を呼ぶと `p` を飛び越えて次の空きマスが返ります。`M` を「末尾より右」を表す番兵として用意します。

|挿入候補 `h`|選ぶ位置|
|---|---|
|`find(h) < M`|`find(h)`|
|`find(h) = M`|先頭へ回り、`find(0)`|

`M-1` の次へ進んだときは番兵に着き、そこで初めて先頭へ回すかを判断します。無理に Union-Find を環状につなぐ必要はありません。

## tips

### 実装

`A` は未使用を `-1` として長さ `M` で初期化します。クエリ `1 x` では `h=x%M` から上の手順で `p` を決め、`A[p]=x` としてから `parent[p]=find(p+1)` に更新します。クエリ `2 x` は `A[x%M]` をそのまま出力します。

親配列は `0..M` の `M+1` 要素にします。経路圧縮ありの `find` を使います。

### よくある誤り

- `x` 自体を添字にする。添字は常に `x mod 2^20` です。
- 末尾まで空きがなければ挿入不能とする。問題では先頭に戻って探索します。
- 埋めた位置の親を更新し忘れる。同じ位置を次の挿入で再び選んでしまいます。

### 計算量

経路圧縮付き Union-Find により、各挿入はならし `O(α(M))`、参照は `O(1)` です。メモリは `O(M)` です。
