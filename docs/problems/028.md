# 028

## 問題リンク

[問題を開く](https://atcoder.jp/contests/abc228/tasks/abc228_d)

## 何に着目するか

ハッシュ表の次の空き位置を線形探索すると、衝突が多い場合に遅くなります。

## 解法方針

各位置から右へ最初に空いている位置を返すUnion-Findを使います。位置を埋めたら、その代表を次の位置の代表へつなぎます。

## 実装要点

末尾の次は先頭に戻るため、番兵としてサイズ `N + 1` の位置を用意します。

## 計算量

各操作はならし `O(α(N))` です。

## 典型・関連問題

典型は **next empty を管理するUnion-Find** です。関連: [ABC217 D](https://atcoder.jp/contests/abc217/tasks/abc217_d)
