# 110

## 問題リンク

[問題を開く](https://atcoder.jp/contests/abc328/tasks/abc328_e)

## 何に着目するか

頂点数が小さく、全域木候補を辺集合の全探索で判定できます。

## 解法方針

辺からN-1本選ぶ組合せを列挙し、Union-Findで閉路がないか確認します。全域木なら重み和のmod Mを最小化します。

## 実装要点

N-1本で閉路がなければ連結です。組合せ生成を使うと実装が簡潔です。

## 計算量

`O(choose(M,N-1) N)` です。

## 典型・関連問題

典型は **小制約での全域木全探索** です。関連: [ABC282 E](058.md)
