# 105

## 問題リンク

[ABC323 E - Playlist](https://atcoder.jp/contests/abc323/tasks/abc323_e)

## キーワード

曲の開始時刻の確率を前から配って、時刻 X を覆う確率を集める

## 何に着目するか

曲が切り替わる瞬間だけを見ると、次に始まる曲は常に一様ランダムです。時刻 `t` に曲が始まる確率を持てば、その曲を選んだ後の開始時刻へ確率を配れます。

## 解法方針

`dp[t]` を「ある曲が時刻 `t` に始まる確率」とします。初期値は `dp[0]=1` です。時刻 `t` から曲 `i` を選ぶと、確率 `1/N` で次の開始時刻 `t+T[i]` へ進みます。

```text
dp[t + T[i]] += dp[t] / N
```

求めるのは曲 1 が時刻 `X` を再生中である確率です。曲 1 が時刻 `t` に始まり、

```text
t ≤ X < t + T[1]
```

なら寄与します。したがって `t` を `max(0, X-T[1]+1)..X` で走査し、`dp[t]/N` を足します（時刻 `t` で曲 1 を選ぶ確率）。

```text
開始 t ───── 曲1の長さ T[1] ─────
           X がこの区間内なら加算
```

## tips

### 実装

必要な開始時刻は高々 `X` なので、DP 配列は `0..X` で十分です。`t+T[i]>X` の遷移は、以後時刻 X を開始時刻として生まないため配らなくて構いません。

確率は `998244353` 上で扱い、`invN` を掛けます。

### よくある誤り

- 時刻 X に曲が始まる確率だけを見る。開始がより前でも再生中なら数えます。
- 曲 1 を選ぶ確率 `1/N` を最後に掛け忘れる。
- `X=t+T[1]` を含める。曲はその時刻に終了して次の曲へ切り替わります。

### 計算量

各時刻で N 曲へ配るため、時間 `O(NX)`、メモリ `O(X)` です。
