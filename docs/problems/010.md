# 010

## 問題リンク

[ABC261 E - Many Operations](https://atcoder.jp/contests/abc261/tasks/abc261_e)

## キーワード

ビット演算の列は、各ビットの 0・1 の写像として合成する

## 何に着目するか

AND・OR・XOR は、あるビットの結果が他のビットに影響しません。大きな整数に対する操作列をそのまま持つ必要はなく、1ビットを入力したときの出力だけを追えば十分です。

## 解法方針

各ビット位置ごとに、これまでの操作列を関数 `f` とみなします。1ビットの入力は0か1しかないため、関数の状態は `f(0), f(1)` の2値で表せます。

| 新しい操作 | `f(0)` と `f(1)` の更新 |
| --- | --- |
| `AND c` | それぞれ `& cの該当ビット` |
| `OR c` | それぞれ `| cの該当ビット` |
| `XOR c` | それぞれ `^ cの該当ビット` |

各クエリ後、現在値 `C` のビットが0なら `f(0)`、1なら `f(1)` を選んで並べれば、操作列をすべて適用した結果になります。

## tips

### 実装

初期状態は恒等関数なので、各ビットで `f(0)=0`, `f(1)=1` です。30ビット程度について独立に更新する実装が素直です。各操作の定数 `c` から `(c >> bit) & 1` を取り出します。

### よくある誤り

- 現在値 `C` にだけ操作を適用する: 次のクエリで必要な「操作列全体」の情報が残りません。
- `f(0)` だけを追う: 入力が1のときの出力も必要です。
- ビットごとに関数を初期化し直す: 関数はクエリをまたいで合成されます。

### 計算量

ビット数を `B` とすると、各操作・各出力は `O(B)` で、全体 `O(NB)` 時間・`O(B)` 空間です。
