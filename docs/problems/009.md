# 009

## 問題リンク

[ABC241 E - Putting Candies](https://atcoder.jp/contests/abc241/tasks/abc241_e)

## キーワード

決定的な遷移で同じ状態を再訪したら、前周期とループに分ける

## 何に着目するか

操作回数 `K` は非常に大きいですが、次に足す値は現在の累積和そのものではなく、その `N` による剰余だけで決まります。取り得る剰余は `N` 種類しかないため、同じ剰余を再び訪れたら、その間の遷移は以後も繰り返されます。

## 解法方針

各剰余 `r` について、初めて到達したときの操作回数と累積和を記録します。再び `r` に来たとき、最初の訪問から今までの区間が1周期です。

| 区間 | 操作回数 | 累積和 |
| --- | ---: | ---: |
| 周期に入る前 | `start` | `sum_start` |
| 1周期 | `len` | `sum_cycle` |
| 残り | `rem` | 記録した遷移を順に加算 |

`K - start` 回を `len` で割り、商の回数だけ `sum_cycle` をまとめて足します。余り `rem` 回だけは、周期中の遷移を先頭からたどります。

## tips

### 実装

`first_step[r]` と `first_sum[r]` に初回情報を保存します。現在の累積和を `x` とすると、次の状態は `x += A[x % N]` です。周期検出前に `K` 回進んだ場合は、そのまま答えにして終了します。

### よくある誤り

- 累積和そのものを状態として扱う: 状態数が無限に増えます。剰余だけで十分です。
- 周期の開始前までを周期に含める: 初回訪問時の操作回数を記録して正確に切り分けます。
- 周期の余りを忘れる: 周期長で割り切れない分は個別に加算します。

### 計算量

高々 `N + K` 回の遷移で、通常は周期を見つけて止まるため `O(N)` 時間・`O(N)` 空間です。
