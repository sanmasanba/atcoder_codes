# 015

## 問題リンク

[ABC254 E - Small d and k](https://atcoder.jp/contests/abc254/tasks/abc254_e)

## キーワード

小さい深さだけをBFSで広げる

## 何に着目するか

クエリごとに始点から距離 `k` 以下の頂点を集めます。グラフ全体を毎回探索すると重そうですが、この問題では `k` が非常に小さいため、遠い頂点まで広げる必要はありません。BFSを距離で打ち切れないか考えます。

## 解法方針

各クエリで始点 `x` からBFSをします。キューには `(頂点, 距離)` を入れ、取り出した頂点の番号を答えに足します。距離が `k` の頂点は答えに含めますが、その先の辺はたどりません。

| 距離 | 行うこと |
| --- | --- |
| `0..k` | 頂点番号を答えへ加算 |
| `0..k-1` | 未訪問の隣接頂点をキューへ追加 |
| `k+1` 以上 | そもそもキューへ入れない |

BFSなので、初めて訪れたときの距離が始点からの最短距離です。訪問済みにすれば、閉路があっても頂点を二重に足しません。

## tips

### 実装

クエリごとに `visited` を作り直し、始点を訪問済みにしてからキューへ入れます。制約が小さいため、全長 `N` の真偽配列を毎回用意しても十分です。

### よくある誤り

- 距離 `k` の頂点を答えに加えない: 条件は「未満」ではなく「以下」です。
- 距離 `k` の頂点からも展開する: 不要な探索になり、制約の利点を失います。
- 訪問済みにするのを取り出し時まで遅らせる: 同じ頂点を何度もキューへ入れます。

### 計算量

1クエリあたり、始点から距離 `k` 以下にある頂点・辺だけに比例します。`k <= 3` という制約の下で十分高速です。
