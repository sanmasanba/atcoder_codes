# 015

## 問題リンク

[問題を開く](https://atcoder.jp/contests/abc254/tasks/abc254_e)

## 何に着目するか

クエリごとに、始点から距離が小さい頂点だけを集めます。探索の深さの上限が小さいことを利用します。

## 解法方針

各クエリで幅優先探索を行い、距離が `k` 以下の頂点の番号を加算します。深さ `k` を超える頂点は展開しません。

## 実装要点

訪問済み配列をクエリごとに用意します。キューには頂点と距離を組で入れます。

## 計算量

1クエリあたり、実際に到達する局所部分に比例します。`k` が小さい制約により十分高速です。

## 典型・関連問題

典型は **深さ制限付きBFS** です。関連: [ABC299 E](https://atcoder.jp/contests/abc299/tasks/abc299_e)
