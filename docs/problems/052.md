# 052

## 問題リンク

[ABC275 E - Sugoroku 4](https://atcoder.jp/contests/abc275/tasks/abc275_e)

## キーワード

反射する移動は、折り返し後の位置を正規化して遷移する

## 何に着目するか

各手での遷移先はサイコロ目ごとに決まり、手数 `K` も小さいため、位置ごとの到達確率をそのまま DP できます。難しいのは `N` を越えたときの反射だけです。

反射は「いったん進んだ位置」を `0..N` へ折り返す関数として先に定義すれば、通常の確率遷移になります。

## 解法方針

`dp[pos]` を現在位置 `pos` にいる確率とします。各手で、`pos<N` からサイコロ目 `d` へ確率 `1/M` を配ります。候補 `q=pos+d` の反射先は、長さ `2N` の周期を使って

```text
t = q mod (2N)
next = t            (t ≤ N)
     = 2N - t       (t > N)
```

と求められます。

```text
0 ─── N ─── 2N
      ↗反射↖
位置 q は [0,N] に折り返される
```

位置 `N` はゴールなので吸収状態です。`dp[N]` は次の手にもそのまま残し、そこからサイコロ遷移は行いません。

## tips

### 実装

確率は `998244353` を法として持ち、`invM = M^(MOD-2)` を掛けます。毎手 `next_dp` をゼロで作り、先に `next_dp[N] += dp[N]` としてから他の位置を遷移させると吸収を明確に書けます。

`q` が `2N` を越えることもあるので、一回だけ `2N-q` とするのではなく、必ず `mod 2N` を取ります。

### よくある誤り

- ゴールからも次の手へ進める。ゴールは吸収状態です。
- `N` を越えたら `2N-q` と一度だけ折り返す。複数回反射するケースがあります。
- 整数除算 `1/M` を使う。法上の逆元です。

### 計算量

`K` 手、各位置 `N+1`、各出目 `M` を処理するため `O(KNM)`、メモリは二層で `O(N)` です。
