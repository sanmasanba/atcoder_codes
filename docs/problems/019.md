# 019

## 問題リンク

[ABC262 D - I Hate Non-integer Number](https://atcoder.jp/contests/abc262/tasks/abc262_d)

## キーワード

選ぶ個数が割る数になる条件は、個数を固定して剰余 DP する

## 何に着目するか

部分集合の和が、その部分集合の要素数で割り切れるかを数えます。要素数が未定のまま剰余を持つと、何を法にすればよいか決まりません。選ぶ個数 `k` を先に固定すれば、必要なのは和を `k` で割った余りだけです。

## 解法方針

各 `k=1..N` について、`dp[j][r]` を「`j` 個選び、和を `k` で割った余りが `r` である選び方」と定義します。要素 `a` を使う遷移は次の通りです。

| 状態 | 遷移先 |
| --- | --- |
| `dp[j][r]` | `dp[j+1][(r+a) mod k]` |

初期状態は `dp[0][0]=1` です。全要素を処理した後の `dp[k][0]` が、ちょうど `k` 個選んで和が `k` の倍数となる通り数です。全ての `k` の答えを足します。

## tips

### 実装

同じ要素を複数回使わない0/1 DPなので、選択数 `j` は大きい方から更新します。`k` ごとに法も剰余配列の長さも変わるため、DP表を作り直します。

### よくある誤り

- 1つのDPで全ての `k` を処理しようとする: 剰余の法が異なります。
- `j` を小さい順に更新する: 同じ要素を同一段階で何度も選んでしまいます。
- `dp[k][0]` 以外を足す: 条件は和が `k` で割り切れることです。

### 計算量

固定した `k` では `O(N^2 k)`、全ての `k` で `O(N^4)` 時間です。1つの `k` のDP表は `O(N^2)` 空間です。
