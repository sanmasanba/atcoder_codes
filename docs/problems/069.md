# 069

## 問題リンク

[ABC232 E - Rook Path](https://atcoder.jp/contests/abc232/tasks/abc232_e)

## キーワード

対称な遷移は、目標との一致・不一致だけに状態を圧縮する

## 何に着目するか

駒がどの正確なマスにいるかを持つと `H×W` 状態になります。しかし、次の一手で目標へ近づく通り数は、目標と行・列が一致しているかだけで決まります。

対称なマスをまとめると、状態は 4 個で済みます。

## 解法方針

目標を `(X,Y)` とし、状態を次の順で持ちます。

|状態|条件|
|---|---|
|`E`|目標マスそのもの|
|`R`|目標と同じ行、異なる列|
|`C`|目標と同じ列、異なる行|
|`N`|行も列も異なる|

たとえば `R` のマスからは、列を目標列へ変える 1 通りで `E`、ほかの列へ変える `W-2` 通りで `R`、行を変える `H-1` 通りで `N` へ遷移します。

```text
R → E : 1
R → R : W-2
R → N : H-1
```

全遷移は次の表です。

|出発|`E` へ|`R` へ|`C` へ|`N` へ|
|---|---:|---:|---:|---:|
|`E`|0|`W-1`|`H-1`|0|
|`R`|1|`W-2`|0|`H-1`|
|`C`|1|0|`H-2`|`W-1`|
|`N`|0|1|1|`H+W-4`|

始点がどの状態かを一つだけ 1 にして、表に従う遷移を `K` 回行います。答えは `dp[E]` です。

## tips

### 実装

各手で長さ 4 の新配列を 0 初期化し、表の係数を掛けて加算します。`H=1` や `W=1` では `H-2` などが負になり得るため、実際には「存在する遷移数」を式のまま用いるか、法の下で正規化します。表の係数を一度小さな盤面で検算します。

すべて `998244353` の法で計算します。

### よくある誤り

- `R` と `C` を一つの状態にまとめる。行移動・列移動の通り数が `H,W` で異なるため区別が必要です。
- 目標マスにいるときにその場へ留まる遷移を入れる。一手では必ず別の行または列へ移動します。
- 初期状態を常に `N` にする。始点と目標が同じ行／列の可能性があります。

### 計算量

各手で定数個の状態を更新するため、時間 `O(K)`、メモリ `O(1)` です。
