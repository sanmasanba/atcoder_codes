# 093

## 問題リンク

[ABC304 E - Good Graph](https://atcoder.jp/contests/abc304/tasks/abc304_e)

## キーワード

連結成分内で同じ扱いなら、成分へ縮約して成分対の制約を持つ

## 何に着目するか

既存の辺で既に連結な頂点どうしは、今後のクエリで区別する必要がありません。禁止条件も、実際には「どの初期連結成分どうしをつないではいけないか」です。

最初に Union-Find で縮約すると、各クエリは根 ID の組が禁止集合にあるかを見るだけになります。

## 解法方針

既存の `M` 辺を Union-Find へ追加します。禁止頂点対 `(x,y)` ごとに `rx=find(x)`, `ry=find(y)` を取り、順序をそろえた `(min(rx,ry),max(rx,ry))` を `bad` 集合へ入れます。

クエリ辺 `(p,q)` も同じ根 ID の組へ変換します。

|根の組|答え|
|---|---|
|`(rp,rq)` が `bad` にある|`No`|
|それ以外|`Yes`|

```text
頂点 --既存辺--> 連結成分（根 ID）
禁止条件         根 ID の組として保存
クエリ             根 ID の組を照合
```

同じ成分内の二頂点なら根の組は `(r,r)` です。禁止対は別成分から来るので通常は `bad` に入らず、`Yes` になります。

## tips

### 実装

禁止集合のキーは順序なし対なので、必ず `a,b=sorted((ra,rb))` のように正規化します。クエリ側にも同じ正規化を行います。

クエリで辺を実際に Union-Find へ追加する問題ではありません。初期成分は全クエリを通して固定です。

### よくある誤り

- クエリごとに Union-Find を併合する。問題は「その辺を追加してよいか」の独立判定です。
- 頂点番号の対をそのまま禁止集合へ入れる。初期成分内の別頂点も同じ意味になります。
- 根 ID の順序をそろえない。`(a,b)` と `(b,a)` を別物としてしまいます。

### 計算量

Union-Find 構築は `O((N+M)α(N))`、禁止対・クエリはそれぞれ期待 `O(α(N))` です。メモリは `O(N+K)` です。
