# 097

## 問題リンク

[ABC314 E - Roulettes](https://atcoder.jp/contests/abc314/tasks/abc314_e)

## キーワード

期待値 DP の自己ループは、式を移項して除去する

## 何に着目するか

残り進捗 `x` から目標までに必要な最小期待コストを `dp[x]` とします。正の出目なら進捗が増えるので、より目標に近い状態だけを参照できます。しかし出目 0 は同じ状態へ戻る自己ループです。

自己ループを式の左辺へ移せば、通常の後ろ向き期待値 DP になります。

## 解法方針

あるルーレットの価格を `C`、出目数を `P`、出目 0 の個数を `z`、正の出目を `s` とします。状態 `x` でこのルーレットを一回選ぶ期待値 `E` は

```text
E = C + (z/P) E + Σ dp[min(M, x+s)] / P
```

です。移項すると

```text
E = (C*P + Σ dp[min(M, x+s)]) / (P-z)
```

|出目|遷移先|
|---|---|
|`0`|同じ `x`（式で移項して消す）|
|`s>0`|`min(M,x+s)`|

各 `x=M-1,...,0` で全ルーレットの `E` を計算し、その最小を `dp[x]` とします。`dp[M]=0` がゴールです。

```text
dp[M] = 0
dp[M-1], dp[M-2], ... を順に確定
```

## tips

### 実装

出目 0 が全てのルーレットは進捗を進められないので候補から除きます。各ルーレットについて正の出目だけを保存しておくと、毎状態での和を計算しやすくなります。

期待値は実数なので `double` を使い、十分小さい誤差で出力します。

### よくある誤り

- 0 の出目を通常の `dp[x]` 参照のまま更新する。未確定状態への循環になります。
- 分母を `P` のままにする。自己ループを移項した後は正の出目数 `P-z` で割ります。
- `x+s>M` を配列外へ遷移する。目標到達後の期待値は `dp[M]=0` です。

### 計算量

ルーレットの出目数総和を `L` とすると、各進捗で全正の出目を調べて `O(ML)`、メモリ `O(M+L)` です。
