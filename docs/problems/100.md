# 100

## 問題リンク

[ABC318 E - Sandwiches](https://atcoder.jp/contests/abc318/tasks/abc318_e)

## キーワード

同じ値を含む組合せは、連続出現の間隔ごとの寄与に分解する

## 何に着目するか

求める三つ組 `(i,j,k)` は `i<j<k` かつ `A[i]=A[k]` です。両端が同じ値である位置対ごとに `j` の選び方を数えることもできますが、全対を列挙すると二乗になります。

同じ値の出現位置を並べ、隣接出現の間をどこで区切るかに注目すると、各三つ組をちょうど一度数えられます。

## 解法方針

ある値の出現位置を `p[0]<p[1]<...<p[m-1]` とします。隣接する `p[t],p[t+1]` の間を、選んだ両端の間に含まれる「最後の隣接境界」として考えます。

左端 `i` は `p[0]..p[t]` の `t+1` 通り、右端 `k` は `p[t+1]..p[m-1]` の `m-t-1` 通り選べます。これらの端点対に対し、`j` は境界の開区間に

```text
p[t+1] - p[t] - 1
```

通りあります。

よって寄与は

```text
(t+1) * (m-t-1) * (p[t+1]-p[t]-1)
```

です。

```text
左端候補 ... p[t] | j の範囲 | p[t+1] ... 右端候補
  t+1 通り          gap          m-t-1 通り
```

全値について隣接出現を走査して足せば答えです。

## tips

### 実装

まず値ごとの出現位置リストを作ります。各リストの長さを `m`、隣接添字を `t` として上式を 64 bit 整数で加算します。

位置は 0-indexed でも差 `p[t+1]-p[t]-1` はそのままです。

### よくある誤り

- 同じ値の全位置対を直接二重ループする。頻度が大きい値で遅くなります。
- `j` の選択数を `p[t+1]-p[t]` とする。両端位置そのものは `j` に使えないので `-1` です。
- 左右の候補数を一方だけ 1 とする。両端は境界の両側のどの同値位置でも選べます。

### 計算量

全リストの長さの合計は `N` なので、時間・メモリともに `O(N)` です。
