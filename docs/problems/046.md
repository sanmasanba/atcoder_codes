# 046

## 問題リンク

[ABC308 F - Vouchers](https://atcoder.jp/contests/abc308/tasks/abc308_f)

## キーワード

購入条件を満たす割引は、価格順に見て最大の得を貪欲に選ぶ

## 何に着目するか

クーポン `(L,D)` は価格が `L` 以上の商品に使えます。商品を価格の昇順に見ると、あるクーポンが使えるようになった後は、以後の商品にもずっと使えます。この単調性により、未使用で使えるクーポンから最大割引を選べばよいと分かります。

## 解法方針

商品価格を昇順にソートし、クーポンを必要価格 `L` の昇順にソートします。今見ている商品価格を `p` とすると、`L≤p` の未登録クーポンをすべて最大ヒープへ追加します。

|商品価格 `p` を処理|ヒープに入れるクーポン|使うもの|
|---|---|---|
|`p`|まだ未登録で `L≤p` の全て|割引額 `D` が最大の一枚|

割引額が最大のクーポンを今の安い商品に使ってよい理由は、今後の商品はさらに高く、現在使えるクーポンはいずれも今後も使えるからです。小さい割引を残して大きい割引を後に回す利点はありません。

全商品価格の和から、選んだ割引額の和を引けば答えです。

```text
価格:       3    5    8
使える D:  {2} {2,4} {2}
選択:       2    4    2
```

## tips

### 実装

クーポンを `(L,D)` でソートし、ポインタ `j` を進めます。Python の `heapq` は最小ヒープなので、`-D` を入れて最大ヒープとして使います。ヒープが空ならクーポンは使いません。

問題では各クーポンを高々一度使えるので、選んだらヒープから pop するだけです。

### よくある誤り

- 商品を価格降順に処理する。`L≤p` のクーポンが後の安い商品では使えなくなり、ヒープの候補集合が単調になりません。
- 使えるクーポンを一枚しかヒープへ入れない。次の商品にも備えて、条件を満たすものを全て追加します。
- 割引額ではなく必要価格が最大のクーポンを選ぶ。目的は支払額最小なので、使える中で `D` 最大です。

### 計算量

ソートに `O(N log N + M log M)`、各クーポンのヒープ操作を含めて `O((N+M) log M)` です。メモリは `O(M)` です。
