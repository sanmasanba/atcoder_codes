# 118

## 問題リンク

[ABC336 E - Digit Sum Divisible](https://atcoder.jp/contests/abc336/tasks/abc336_e)

## キーワード

桁和を先に固定して「和」と「剰余」を同時に持つ桁 DP を回す

## 何に着目するか

Harshad 数の条件は「数が自分の桁和で割り切れる」です。桁和が未知のまま剰余を持てませんが、桁和 `s` を固定すれば、`mod s` の通常の桁 DP になります。

## 解法方針

`s=1..9×桁数` を全探索します。固定した `s` について

```text
dp[pos][sum][rem][tight]
```

を「先頭から `pos` 桁決め、桁和 `sum`、`s` での余り `rem`、上限一致状態」とします。次の桁を `d` とすると

```text
next_sum = sum + d
next_rem = (rem*10 + d) mod s
```

最後に `sum=s` かつ `rem=0` の状態を加えます。

```text
桁和 s を固定
  ├ 合計桁和を s に合わせる
  └ 数値を s で割った余りを 0 に合わせる
```

各正整数は桁和を一つだけ持つため、異なる `s` の集計で二重計数されません。

## tips

### 実装

上限 `N` の桁列を左から処理します。先頭 0 を許す DP にすると、桁数の短い数も同時に数えられます。ただし数 0 は対象外なので、最終集計では自然に `s≥1` の条件により除かれます。

`sum>s` の状態は遷移させません。`tight` は現在桁の上限を `digit[pos]` か 9 かに切り替えます。

### よくある誤り

- 一つの DP で全ての剰余法を扱おうとする。法が桁和自身なので s 固定が必要です。
- `sum=s` だけを確認して余りを見ない。割り切れ条件も必要です。
- `tight` を外し、N を超える数を数える。

### 計算量

桁数を `L` とすると、`s` の総和を含めておおむね `O(L^4)` 程度です。制約の桁数では十分です。
