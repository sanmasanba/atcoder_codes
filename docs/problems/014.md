# 014

## 問題リンク

[ABC253 E - Distance Sequence](https://atcoder.jp/contests/abc253/tasks/abc253_e)

## キーワード

DP 遷移先が連続区間なら、累積和で区間和に置き換える

## 何に着目するか

末尾の値を決めるDPでは、前の値を全て試すと1段あたり `O(M^2)` になります。しかし「差が `K` 以上」という条件で使える前の値は、左側と右側の2区間です。連続区間の和を一度に取れないか考えます。

## 解法方針

`dp[i][x]` を長さ `i`、末尾が `x` の数列の個数とします。前の行の累積和 `prefix` を作れば、任意の区間 `[l, r]` の和を `prefix[r] - prefix[l-1]` で求められます。

次に `x` を置ける前の値は次の2区間です。

| 区間 | 条件 |
| --- | --- |
| `[1, x-K]` | `前の値 <= x-K` |
| `[x+K, M]` | `前の値 >= x+K` |

この2区間の和を足して `dp[i][x]` とします。範囲外の区間は空として0にします。

## tips

### 実装

各段で `prefix[0]=0` とし、`prefix[x]=prefix[x-1]+dp[x]` を法で計算します。次の行だけが必要なので、DP配列はローリング配列にできます。

### よくある誤り

- `x-K` や `x+K` をそのまま添字にする: `1..M` に収めます。
- 禁止区間を引く方法で端を取り違える: まず許可区間を2つ書き出すと安全です。
- `K=0` を特別扱いし忘れる: 全ての前の値が使えます。

### 計算量

各段で累積和と `M` 個の遷移を行うため `O(NM)` 時間、ローリング配列で `O(M)` 空間です。
