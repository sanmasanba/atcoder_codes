# 036

## 問題リンク

[ABC220 F - Distance Sums 2](https://atcoder.jp/contests/abc220/tasks/abc220_f)

## キーワード

根を変えた全頂点の値は、親子間の差分で再根 DP する

## 何に着目するか

各頂点を根にして全頂点までの距離和を DFS で計算すると `O(N^2)` です。しかし根を親 `v` から子 `to` へ一辺だけ動かしたとき、距離が変わるのは「子の部分木に入るか」だけで分類できます。

## 解法方針

まず根を `0` に固定して DFS し、各頂点の部分木サイズ `size[v]` と `answer[0]`（根 0 からの距離和）を求めます。

次に、親 `v` の答えから子 `to` の答えを計算します。

|頂点の場所|個数|根を `v→to` へ移したときの距離変化|
|---|---:|---:|
|`to` の部分木内|`size[to]`|`-1`|
|それ以外|`N-size[to]`|`+1`|

よって次の式が成り立ちます。

```text
answer[to] = answer[v] - size[to] + (N - size[to])
           = answer[v] + N - 2*size[to]
```

根 0 の答えが一つ分かれば、この式を親から子へ伝播して全頂点の答えを `O(N)` で得られます。

```text
1 回目 DFS: size と answer[0]
2 回目 DFS: answer[parent] → answer[child]
```

## tips

### 実装

1 回目では、深さを加算して `answer[0]` を作り、子から戻った後に `size[v] += size[to]` とします。`size[v]` の初期値は自分自身を含む `1` です。

2 回目では、親を引数に渡して逆向きの辺を除きます。再帰が深い場合は反復 DFS を使うか、再帰上限を上げます。

### よくある誤り

- `size[to]` から子 `to` 自身を除く。距離が 1 近くなるのは子自身も含めた部分木全体です。
- `+N` と `-2*size[to]` の符号を逆にする。子側は近くなり、それ以外は遠くなります。
- 1 回目の DFS で深さを足し忘れる。初期の距離和が必要です。

### 計算量

二回の DFS で、時間・メモリともに `O(N)` です。
