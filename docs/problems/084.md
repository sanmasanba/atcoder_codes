# 084

## 問題リンク

[ABC280 D - Factorial and Multiple](https://atcoder.jp/contests/abc280/tasks/abc280_d)

## キーワード

階乗の倍数判定は、素因数ごとの指数を数えて二分探索する

## 何に着目するか

`K | N!` となる最小の `N` を求めます。`K` を素因数分解して `K=∏p^e` とすると、全ての素数 `p` について `N!` に `p` が少なくとも `e` 個含まれればよいです。

一つの素数について必要な `N` を求め、その最大値を取れば全素数条件を満たします。

## 解法方針

`N!` に含まれる素数 `p` の指数は Legendre の公式で

```text
v_p(N!) = floor(N/p) + floor(N/p²) + floor(N/p³) + ...
```

です。`v_p(N!)` は `N` に対して単調非減少なので、`v_p(N!)≥e` となる最小 `N` を二分探索できます。

|項|数えているもの|
|---|---|
|`N/p`|p の倍数が少なくとも 1 個持つ p|
|`N/p²`|p² の倍数が追加で持つもう 1 個の p|
|`N/p³`|さらに追加の p|

各素数の最小値を `need[p]` とすると、答えは `max_p need[p]` です。

## tips

### 実装

まず `K` を試し割りで素因数分解します。`count_p(n,p)` は `n//=p` を繰り返して和へ足すと、べき乗を作らずに書けます。

二分探索の上界は `K` で十分です（`K | K!`）。`K` が大きくても 64 bit 整数で扱います。

### よくある誤り

- `N!` の p の個数を `N/p` だけにする。p², p³ の倍数の余分な p を忘れます。
- 各素数に対する最小 `N` を足す。すべて同じ `N!` に含まれればよいので最大です。
- 素因数分解後の残りの大きな素数を処理し忘れる。

### 計算量

試し割りは `O(√K)`、各素数での二分探索と指数計算は対数回です。追加メモリは素因数の個数分です。
