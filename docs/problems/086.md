# 086

## 問題リンク

[ABC290 E - Make it Palindrome](https://atcoder.jp/contests/abc290/tasks/abc290_e)

## キーワード

全区間の総数は、全体の寄与から条件を満たす位置対の寄与を引く

## 何に着目するか

各部分配列を回文にするための変更回数は、対称位置で値が異なる組の数です。最初に全ての部分配列について「対称位置の組数」を数え、同じ値で既に一致している組の寄与を値ごとに引きます。

## 解法方針

長さ `L` の部分配列には `floor(L/2)` 個の対称位置対があります。これを全長について足したものを `total` とします。

次に、同じ値の出現位置リストを使って、一致する対称位置の寄与を数えます。位置 `i<j` がある部分配列内で対称になるには、部分配列の左端と右端が `i,j` の外側に同じだけ広がる必要があります。この選び方を、同じ値の出現位置を左から順に処理しながら集計します。

```text
部分配列の対称位置
left ... i ... j ... right
          A[i] = A[j] なら変更不要
```

値ごとの位置列に対し、左側の出現位置数と位置和を累積すると、各右側位置への一致寄与を一括で求められます。`total - matched` が答えです。

## tips

### 実装

各値の出現位置を配列に集めます。位置列を走査し、現在位置との距離・左右へ拡張できる数を式にして加算します。答えは `O(N^3)` になり得るため 64 bit 整数を使います。

全対称位置数 `total` は、各長さごとに数えるか、右端を固定して `floor((r-l+1)/2)` の和を閉形式で計算します。

### よくある誤り

- 異なる値の位置対を直接数える。同じ値で一致する寄与を引くほうが値ごとの集計にできます。
- 対称位置対の数を部分配列長 `L` とする。必要なのは `floor(L/2)` です。
- 同じ値の全位置対を二重ループする。頻度が大きい値で二乗になるため累積和でまとめます。

### 計算量

位置列ごとの累積集計により、全体で `O(N)` または `O(N log N)` 程度、メモリ `O(N)` です。
