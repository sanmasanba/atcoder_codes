# 103

## 問題リンク

[ABC321 E - Complete Binary Tree](https://atcoder.jp/contests/abc321/tasks/abc321_e)

## キーワード

木で距離 K の頂点は、祖先ごとの反対側部分木に分けて数える

## 何に着目するか

頂点番号 `v` の子は `2v,2v+1`、親は `floor(v/2)` の暗黙完全二分木です。頂点 `X` から距離 `K` の経路は、最初に上へ何辺進むかで一意に分類できます。

ある祖先まで上がった後に下るなら、来た子側へ戻ることはできないので「反対側の子の部分木」だけを数えます。

## 解法方針

`count_desc(v,d)` を「`v` の子孫で、`v` からちょうど深さ `d` にあり、番号が `N` 以下の頂点数」とします。これは区間

```text
[v*2^d, (v+1)*2^d - 1]
```

と `1..N` の共通部分の長さです。

距離 `K` の頂点は次を足します。

1. `X` の真下へ `K` 進む: `count_desc(X,K)`
2. `i=1..K` について、`X` から `i` 回上がった祖先 `p` で止まる:
   - `i==K` なら祖先 `p` 自身を 1 個加える
   - `i<K` なら、`p` の「`X` 側でない子」の深さ `K-i-1` の子孫を加える

```text
X から上へ i 回 → p
                 └ 反対側の子 → 残り K-i-1 段下
```

祖先が根の上へ出たら、その先は存在しないので打ち切ります。

## tips

### 実装

親へ上がる途中で「一つ前にいた子」を `child` として覚えます。祖先 `p` の反対側の子は `2*p` と `2*p+1` のうち `child` でないほうです。

`2^d` や区間端は大きくなるので、`N` を超えた時点で打ち切るか 64 bit 整数を使います。

### よくある誤り

- 祖先から下るときに来た側の部分木も数える。そこへ行く経路は一度上った辺を戻るため、単純距離が短くなります。
- `i==K` の祖先自身を忘れる。
- `X` の下方向だけを数える。祖先方向とその枝も距離 K に含まれます。

### 計算量

祖先を高々 `O(log N)` 回たどり、各部分木数は定数時間で求められます。各クエリ `O(log N)`、追加メモリ `O(1)` です。
