# 083

## 問題リンク

[ABC279 F - BOX](https://atcoder.jp/contests/abc279/tasks/abc279_f)

## キーワード

名前と実体の対応が動く集合は、識別子と集合ノードを分離する

## 何に着目するか

箱を丸ごと別箱へ移す・新しいボールを入れる・ボールが入る箱を聞く操作があります。箱の中身を実際に移すと遅いので、箱が指す「集合実体の根」と、各ボールが属するノードを分離します。

## 解法方針

Union-Find 風の親ポインタを、ボール／集合ノードから「箱を表す根」へ向けます。各箱 `b` について現在の実体根 `root_of_box[b]` を、根ノードについて箱番号 `box_of_root[root]` を持ちます。

|操作|ポインタ操作|
|---|---|
|箱 y の中身を箱 x へ|`root_y` を `root_x` へつなぎ、y は空にする|
|新ボールを箱 x へ|新ノードを作り `root_x` へつなぐ|
|ボール b の箱を問う|`find(ball_node[b])` の `box_of_root`|

箱 x が空の場合は、新しい空実体ノードを x 用に作ってから併合／追加します。根を残す向きに親を張ると、箱番号の対応を更新しやすくなります。

## tips

### 実装

操作数ぶん新ノードが増えるので、初期ボール `N` に `Q` を足した配列を用意します。空箱を表すノードも必要です。`find` は経路圧縮します。

### よくある誤り

- 箱 y の各ボールを走査して移す。大量の移動で遅くなります。
- 箱 y を空にした後の実体ノードを用意しない。後で y にボールを入れられません。
- 根に対応する箱番号を更新しない。問い合わせで箱名を復元できません。

### 計算量

各操作は Union-Find の find / 親付けだけなので、ならし `O(α(N+Q))`、メモリ `O(N+Q)` です。
