# 004

## 問題リンク

[ABC260 C - Changing Jewels](https://atcoder.jp/contests/abc260/tasks/abc260_c)

## キーワード

再帰的な変換は、変換先の個数を状態にして下から計算する

## 何に着目するか

宝石を変換すると、必ずレベルが1つ下がります。あるレベルの宝石が最終的に何個の青レベル1宝石になるかは、より下のレベルの答えだけから決まります。変換を再帰的に展開する代わりに、レベルごとの結果を下から表にできないか考えます。

## 解法方針

`red[l]` を「赤レベル `l` 宝石1個から得られる青レベル1宝石の個数」、`blue[l]` を青レベル `l` 宝石1個から得られる個数とします。

レベル1では、赤は青レベル1に変換され、青レベル1はそのままなので、どちらも `1` です。レベル `l (>= 2)` の変換規則をそのまま式にすると、

```text
blue[l] = red[l - 1] + Y × blue[l - 1]
red[l]  = red[l - 1] + X × blue[l]
```

となります。`blue[l]` を先に求めてから `red[l]` を求める順序に注意します。

| 状態 | 意味 | 参照する状態 |
| --- | --- | --- |
| `blue[l]` | 青レベル `l` 1個の最終結果 | `red[l-1]`, `blue[l-1]` |
| `red[l]` | 赤レベル `l` 1個の最終結果 | `red[l-1]`, **`blue[l]`** |

最終的に必要なのは、最初の赤レベル `N` 宝石なので `red[N]` です。

## tips

### 実装

`red` と `blue` を長さ `N + 1` の配列にし、`1` から順に埋めます。各レベルは直前のレベルだけを見るため、配列を使わず2組の変数に圧縮しても構いません。

### よくある誤り

- `red[l]` を `blue[l]` より先に計算する: 同じレベルの `blue[l]` を使う遷移です。
- 赤・青のどちらかだけを状態にする: 変換先に両方の色が現れるため、必要な情報が不足します。
- レベル1の基底を0にする: 最終的に数える青レベル1宝石は1個です。

### 計算量

各レベルを1回ずつ処理するので `O(N)` 時間です。配列なら `O(N)`、変数に圧縮すれば `O(1)` 空間です。
