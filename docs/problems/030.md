# 030

## 問題リンク

[ABC240 E - Ranges on Tree](https://atcoder.jp/contests/abc240/tasks/abc240_e)

## キーワード

木の葉へ連番を振ると、各部分木を葉番号の区間として表せる

## 何に着目するか

葉へ DFS で連番を振ると、ある頂点の部分木に含まれる葉は必ず連続番号になります。子の部分木を順にすべて探索してから次の子へ移るためです。

各頂点について欲しい区間は、子の区間をまとめれば求まります。木を下から上へ情報を返す DFS そのものです。

## 解法方針

根を `0` として親を決めます。DFS 中に葉へ到達したら、葉カウンタを 1 増やし、その値を `[L,R]=[cnt,cnt]` とします。内部頂点なら、全ての子から返る区間を集約します。

```text
        v
      /   \
   子1      子2
 [2,4]    [5,7]

v の区間 = [ min(2,5), max(4,7) ] = [2,7]
```

DFS 順に子を訪れるので、実際には子の区間は互いに隣接します。そのため左端は最初の子の `L`、右端は最後の子の `R` でも求められますが、`min/max` で書くと意図が分かりやすくなります。

## tips

### 実装

DFS 関数が `(L,R)` を返すようにするか、子を処理した後に `L[v]`, `R[v]` を更新します。親 `p` を引数に渡し、隣接リストの `to == p` を飛ばせば、無向辺をそのまま使えます。

根だけは次数 0 でない限り葉ではありません。一般に「親以外の子が 0 個」なら葉、という判定にすると根も一貫して扱えます。

### よくある誤り

- 全頂点に DFS 番号を振る。この問題で番号を振る対象は葉だけです。
- 根を次数 1 だから葉と扱う。`N>1` の根は子を持つので内部頂点です。
- 子の処理前に親の区間を確定する。部分木の葉をすべて見終えてから決まります。

### 計算量

各頂点・辺を一度ずつ見るため、時間・メモリともに `O(N)` です。
