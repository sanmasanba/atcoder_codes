# 090

## 問題リンク

[ABC298 E - Unfair Sugoroku](https://atcoder.jp/contests/abc298/tasks/abc298_e)

## キーワード

交互に進む確率ゲームは、両者の位置と手番で期待値 DP する

## 何に着目するか

盤面上の二人の位置が分かれば、以後の勝率は過去の経路に依存しません。高橋が先にゴールする確率を、二人の位置と手番の状態として持ちます。

ゴールした側は以後勝敗が確定する吸収状態です。

## 解法方針

`T[a][b]` を「高橋の手番で、高橋が a、青木が b にいるときの高橋勝率」、`A[a][b]` を青木の手番版とします。

終端条件は

```text
T[N][b] = A[N][b] = 1       # 高橋が先に到着
T[a][N] = A[a][N] = 0       # 青木が先に到着
```

高橋番では出目 `d=1..P` を平均して青木番へ、青木番では `d=1..Q` を平均して高橋番へ遷移します。

```text
T[a][b] = (1/P) Σ A[min(N,a+d)][b]
A[a][b] = (1/Q) Σ T[a][min(N,b+d)]
```

遷移先は少なくとも片方の位置が増えるので、位置を大きい方から小さい方へ埋めれば参照先は確定済みです。

```text
高橋番 → 高橋位置が増える
青木番 → 青木位置が増える
```

初期状態 `T[A][B]` が答えです。

## tips

### 実装

確率は `998244353` 上で `invP`,`invQ` を掛けます。ゴールを超えた位置は必ず `N` へ丸めます。

二つの二次元配列を用意するか、手番を一つの次元にして持ちます。

### よくある誤り

- ゴールを超えた位置をそのまま配列外にする。到達は `N` と同じです。
- 青木が到着した後も高橋番へ遷移する。勝敗は即確定です。
- 手番を状態へ入れず、二人の遷移を同じ式で混ぜる。

### 計算量

全位置対で各出目を調べるため、時間 `O(N^2(P+Q))`、メモリ `O(N^2)` です。
