# 087

## 問題リンク

[ABC292 E - Transitivity](https://atcoder.jp/contests/abc292/tasks/abc292_e)

## キーワード

推移閉包の辺数は、各始点からの到達数から直接辺を除いて数える

## 何に着目するか

欲しいのは「辺が直接なくても、長さ 2 以上の有向パスで到達できる順序付き頂点対」です。始点を一つ固定し、そこから到達できる頂点を DFS/BFS すれば、その始点から追加できる辺候補が分かります。

この問題の制約では、全頂点を始点にして探索しても間に合います。複雑な到達可能性データ構造は不要です。

## 解法方針

各頂点 `s` を始点として DFS/BFS を行い、到達できた頂点数を `reach` とします。`s` から直接出ている辺の数を `outdeg[s]` とすると、`s` を始点とする答えへの寄与は

```text
reach - 1 - outdeg[s]
```

です。

|項|意味|
|---:|---|
|`reach`|始点自身を含む到達可能頂点数|
|`-1`|`(s,s)` は数えない|
|`-outdeg[s]`|既に直接辺がある対は数えない|

```text
s → a → b
s → c

到達先 {s,a,b,c} から s と直接辺 a,c を除く
→ 新たに必要な関係は (s,b)
```

全 `s` の寄与を足せば答えです。

## tips

### 実装

各始点ごとに `visited` を全て false に戻し、スタックまたはキューで探索します。到達頂点数は訪問時に数えます。

自己ループがない制約でも、始点は必ず visited にするため `-1` を引く理由が明確です。

### よくある誤り

- 到達可能対をそのまま答えにする。既存の直接辺は追加不要です。
- 始点自身を数える。パス長 0 の関係は対象外です。
- 一回の BFS だけで全始点の答えが分かると考える。到達可能性は始点ごとに異なります。

### 計算量

各始点で `O(N+M)` の探索をするため、全体 `O(N(N+M))`、メモリは `O(N+M)` です。
