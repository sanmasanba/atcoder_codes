# 108

## 問題リンク

[ABC326 E - Revenge of "The Salary of AtCoder Inc."](https://atcoder.jp/contests/abc326/tasks/abc326_e)

## キーワード

各番号が支給される到達確率を prefix 和で更新する

## 何に着目するか

`A[i]` の期待値への寄与は、手順中に番号 `i` が選ばれる確率 `p[i]` を掛けたものです。番号 `i` へ進めるのは、それより小さい番号 `j` にいる状態から次のダイスで i が出るときだけです。

## 解法方針

便宜上 `p[0]=1` とします。番号 `i` が支給される確率は

```text
p[i] = (p[0] + p[1] + ... + p[i-1]) / N
```

です。各状態 j から直接 i を出す確率が `p[j]/N` だからです。prefix 和を `S` とすると、左から

```text
p[i] = S / N
S += p[i]
answer += A[i] * p[i]
```

と更新できます。

```text
0, 1, ..., i-1 の到達確率
          ↓ 次の出目が i
       p[i] を作る
```

## tips

### 実装

`invN=N^(MOD-2)` を前計算し、除算は `invN` を掛けます。初期 `S=1` は `p[0]` を表します。

### よくある誤り

- 状態 i からも i へ進めるとする。`x<y` が必要なので同じ出目で終了します。
- 期待値を状態 DP で二重に数える。各 `A[i]` の支給確率を一度求めれば足し合わせるだけです。

### 計算量

時間 `O(N)`、追加メモリ `O(1)` です。
