# 042

## 問題リンク

[ABC295 D - Three Days Ago](https://atcoder.jp/contests/abc295/tasks/abc295_d)

## キーワード

区間内の偶奇条件は、prefix XOR のビットマスクの一致で数える

## 何に着目するか

ある部分文字列で全ての数字の出現回数が偶数であることは、各数字の「偶奇」だけで判定できます。数字 `0..9` の偶奇を 10 ビットで持つと、文字を一つ読む操作は対応ビットの XOR になります。

区間の偶奇ベクトルは、両端の prefix の XOR です。全ビットが 0 である条件は、両端のマスクが等しいことと同値です。

## 解法方針

`mask[i]` を先頭から `i` 文字読んだときの偶奇マスクとします。数字 `d` を読んだら `mask ^= 1<<d` でその数字の偶奇を反転します。

```text
prefix l の mask = 010101...
prefix r の mask = 010101...
        ↓ XOR
区間 [l,r) の偶奇 = 000000...  （全数字が偶数回）
```

右端を左から順に動かし、現在マスクと同じマスクが過去に何回出たかを `count[mask]` で数えます。その個数だけ、現在の右端を終点とする条件を満たす区間があります。

|処理順|操作|
|---|---|
|初期|空 prefix の `count[0]=1`|
|数字 `d` を読む|`mask ^= 1<<d`|
|答えへ加算|`count[mask]`|
|最後|`count[mask] += 1`|

## tips

### 実装

マスクは 10 ビットなので、辞書ではなく長さ `1<<10` の配列で十分です。答えは部分文字列数で最大 `O(N^2)` になるため `long long` / Python `int` を使います。

現在マスクを count に足す前に、過去の同じマスク数を答えへ加算します。順序を逆にすると空でない区間以外の扱いが崩れます。

### よくある誤り

- 各数字の出現回数そのものを状態にする。必要なのは偶奇だけです。
- `count[0]=1` を入れない。先頭から始まる有効な部分文字列を数え落とします。
- XOR ではなくビット OR を使う。偶奇は二回出ると元に戻ります。

### 計算量

文字列を一度走査するだけで、時間は `O(N)`、メモリは `O(2^{10})` です。
