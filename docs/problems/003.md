# 003

## 問題リンク

[ABC249 D - Index Trio](https://atcoder.jp/contests/abc249/tasks/abc249_d)

## キーワード

積や和の条件が値だけで決まるなら、添字を頻度に集約する

## 何に着目するか

`(i, j, k)` を直接選ぶと `N^3` 通りあります。一方、条件は添字そのものではなく `A_i × A_j = A_k` という値だけで決まります。同じ値を持つ要素は区別せず、値ごとに何個あるかを先に集計できないか考えます。

## 解法方針

値 `v` の出現回数を `cnt[v]` とします。`a × b = c` を満たす値の組を1つ固定すると、`A_i = a`, `A_j = b`, `A_k = c` となる添字の選び方は

```text
cnt[a] × cnt[b] × cnt[c]
```

通りです。これをすべての `a, b` について足せば答えになります。

`c = a × b` は `a, b` が決まれば一意です。そのため `a` と `b` を走査し、`a × b` が配列の最大値を超えたら打ち切れます。

| 固定する値 | 出現回数 | 選び方 |
| --- | ---: | ---: |
| `a` | `cnt[a]` | `i` の選び方 |
| `b` | `cnt[b]` | `j` の選び方 |
| `a × b` | `cnt[a × b]` | `k` の選び方 |

掛け合わせるのは、3つの添字を独立に選べるからです。`a = b` や `a × b = a` の場合でも、同じ添字を選ぶことが許されているため、この式をそのまま使えます。

## tips

### 実装

配列の最大値を `M` とし、長さ `M + 1` の頻度配列を作ります。`a` を1から `M` まで動かし、`b` は `a × b <= M` の範囲だけ調べます。`cnt[a]` または `cnt[b]` が0なら、その組は飛ばせます。

### よくある誤り

- `i < j < k` として数える: この問題は順序つき三つ組なので、添字間の大小は不要です。
- 値が同じときに組合せを使う: 同じ添字を選べるので、`cnt[v]^2` や `cnt[v]^3` をそのまま使います。
- `a × b` の値が配列の範囲外なのに頻度配列を参照する: 最大値を超えた時点で打ち切ります。

### 計算量

`a` ごとに `M / a` 程度の `b` を調べるので、合計は `O(M log M)` です。頻度配列は `O(M)` 空間です。
