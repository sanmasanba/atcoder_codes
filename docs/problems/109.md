# 109

## 問題リンク

[ABC327 E - Maximize Rating](https://atcoder.jp/contests/abc327/tasks/abc327_e)

## キーワード

選択数ごとの減衰加重和を DP し、分母の異なる評価式を最後に比較する

## 何に着目するか

選んだコンテストの得点は、新しいものほど大きい重みを持つ減衰和になります。一方でペナルティ `1200/sqrt(k)` と分母は選択数 `k` に依存します。選択数を混ぜて比較せず、`k` ごとに最良の分子を DP します。

## 解法方針

`dp[k]` を「これまで見たコンテストからちょうど `k` 個選んだときの、順序を反映した最大加重和」とします。新しいレート `P` を最後に選ぶと、既存の加重和は係数 `0.9` 倍され、新しい `P` が足されます。

```text
dp[k+1] = max(dp[k+1], 0.9 * dp[k] + P)
```

`k` 個選んだ場合の分母は

```text
den[k] = 1 + 0.9 + 0.9² + ... + 0.9^(k-1)
```

なので最終評価は

```text
dp[k] / den[k] - 1200 / sqrt(k)
```

です。全ての `k≥1` で最大を取ります。

|選択|加重和への影響|
|---|---|
|選ばない|`dp[k]` を維持|
|新しい P を最後に選ぶ|`0.9*dp[k] + P`|

## tips

### 実装

同じコンテストを複数回選ばないため、`k` は大きい方から降順に更新します。`dp` の未到達状態は十分小さい負値にします。

`den[k]` は前計算しておくと、最後の評価が簡単です。浮動小数点で計算し、出力精度に注意します。

### よくある誤り

- `dp[k+1]=0.9*P+dp[k]` とする。新しいコンテストが重み 1、過去が 0.9 倍です。
- 選択数を状態に入れず、途中で評価式を比較する。分母とペナルティが異なります。
- `k` を昇順に更新する。同じ `P` を同一ループで繰り返し選べます。

### 計算量

`N` 個について選択数を更新するため時間 `O(N^2)`、メモリ `O(N)` です。
