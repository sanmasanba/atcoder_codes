# 101

## 問題リンク

[ABC319 E - Bus Stops](https://atcoder.jp/contests/abc319/tasks/abc319_e)

## キーワード

時刻依存の遷移が小さな周期を持つなら、周期ごとに前計算する

## 何に着目するか

バスを待つ時間は、到着時刻を各バス周期 `P[i]` で割った余りだけで決まります。全周期の最小公倍数 `L` で時刻をずらすと、全ての待ち時間が同じになり、到着時刻も `L` だけずれます。

この問題では `L` が小さいため、出発時刻の余り全てを前計算できます。

## 解法方針

徒歩時間 `X` の後に出発する時刻を `t` とします。`r=0..L-1` それぞれについて、時刻 `cur=r` から全バスを順にシミュレーションし、最終到着までの追加時間 `add[r]=cur-r+Y` を保存します。

バス `(P,T)` では、次の出発までの待ち時間は

```text
wait = (-cur) mod P
cur += wait + T
```

クエリ開始時刻を `q` とすると、答えは

```text
q + X + add[(q+X) mod L]
```

```text
q --徒歩 X--> バス区間（余りだけで決まる） --徒歩 Y--> 到着
```

## tips

### 実装

`L` は全 `P[i]` の lcm です。この問題の制約では最大でも十分小さい値になります。余り計算のため、`cur` は 64 bit 整数で持ちます。

`(-cur) % P` は Python ではそのまま非負の待ち時間になります。C++ では `(P-cur%P)%P` と書くと安全です。

### よくある誤り

- クエリ時刻 `q` の余りをそのまま使う。最初の徒歩 `X` の後にバスへ乗ります。
- `L` を周期の積にする。最小公倍数でよく、積では大きくなりすぎます。
- 待ち時間を `P-cur%P` とし、ちょうど時刻一致のとき `P` 待つ。0 待ちです。

### 計算量

前処理は `O(LN)`、各クエリは `O(1)`、メモリは `O(L)` です。
