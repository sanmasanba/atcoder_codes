# 038

## 問題リンク

[ABC254 F - Rectangle GCD](https://atcoder.jp/contests/abc254/tasks/abc254_f)

## キーワード

行列が行・列の和で表せるなら、差分の GCD に分解する

## 何に着目するか

長方形の各マスの値は `A[i]+B[j]` です。全マスの GCD を直接取る必要はありません。左上の一値を基準にして、行・列方向の変化量（隣接差分）の GCD だけを集めれば、長方形中の全値の GCD が復元できます。

## 解法方針

`DA[i]=A[i+1]-A[i]`、`DB[j]=B[j+1]-B[j]` とします。クエリ範囲を行 `[h1,h2]`、列 `[w1,w2]` としたとき、基準は左上の `A[h1]+B[w1]` です。

範囲内の他の値は、基準値に `DA` の区間和と `DB` の区間和を足した形です。GCD は加減算で不変なので、答えは次になります。

```text
gcd(
  A[h1] + B[w1],
  gcd(DA[h1..h2-1]),
  gcd(DB[w1..w2-1])
)
```

|長方形内の移動|基準値から増減する量|
|---|---|
|一行下へ|`DA` の和|
|一列右へ|`DB` の和|

`DA`, `DB` の区間 GCD を得るために、それぞれセグメント木（または Sparse Table）を作ります。

## tips

### 実装

差分は負になり得ますが、`gcd` には絶対値を渡すか、言語の `gcd` が符号を正規化することを確認します。行または列の幅が 1 のとき、対応する差分区間は空で、その GCD は `0` とみなします。

Python の `math.gcd(x, 0)` は `abs(x)` なので、空区間を `0` にすると式を分岐なしで書けます。

### よくある誤り

- `A` と `B` 本体の区間 GCD を取る。必要なのは隣接値の**差分**です。
- `h2` や `w2` を差分区間に含める。要素数 `k` の範囲に差分は `k-1` 個です。
- 左上ではなく任意の値を基準にして、差分の対応範囲をずらす。

### 計算量

差分配列とセグメント木の構築は `O(N+M)`、各クエリは GCD 区間取得二回で `O(log(N+M))` です。メモリは `O(N+M)` です。
