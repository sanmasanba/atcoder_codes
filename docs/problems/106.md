# 106

## 問題リンク

[ABC324 E - Joint Two Strings](https://atcoder.jp/contests/abc324/tasks/abc324_e)

## キーワード

前文字列で作れる prefix 長と後文字列で作れる suffix 長を足して判定する

## 何に着目するか

`S[i]+S[j]` に目標文字列 `T` が部分列として現れるとき、`T` の前半を `S[i]`、後半を `S[j]` から取れます。どこで分けるかを全て試す代わりに、各文字列が前から・後ろから何文字一致できるかだけを求めます。

## 解法方針

各文字列 `S` について次を計算します。

|値|意味|
|---|---|
|`pre[S]`|`S` を左から使って `T` の prefix を作れる最大長|
|`suf[S]`|`S` を右から使って `T` の suffix を作れる最大長|

連結 `S[i]+S[j]` が条件を満たす必要十分条件は

```text
pre[S[i]] + suf[S[j]] ≥ |T|
```

です。`pre` の値を昇順ソートしておき、各 `suf` に対して `|T|-suf` 以上の `pre` の個数を二分探索で数えます。

```text
T = [ prefix ] [ suffix ]
       ↑ S[i]     ↑ S[j]
```

前後の一致長は、`T` と `S` を同時に一度走査すれば得られます。suffix は両方を反転して同じ処理にしても構いません。

## tips

### 実装

`pre` は `T` のポインタを 0 から進め、同じ文字を見つけたら増やします。`suf` は末尾から同様に行います。

二分探索は `lower_bound(pre_values, len(T)-suf)` から末尾までの個数です。必要長が 0 以下なら全ての前文字列が使えます。

### よくある誤り

- `pre+pre` や `suf+suf` を比較する。前側と後側は別の文字列です。
- 部分列を部分文字列として連続一致で数える。途中の文字は飛ばせます。
- `i==j` を除く。この問題では同じ文字列を二回選んでもよいです。

### 計算量

全文字列長を `L` とすると一致長計算は `O(L)`、ソートと二分探索を含めて `O(L+N log N)`、メモリ `O(N)` です。
