# 111

## 問題リンク

[ABC329 E - Stamp](https://atcoder.jp/contests/abc329/tasks/abc329_e)

## キーワード

消去済みをワイルドカードとして、可能になった局所一致をキューで連鎖させる

## 何に着目するか

長さ `M` の区間を消せるのは、各文字が `T` と一致するか、すでに消えてワイルドカード `#` になっているときです。一度消えた文字は以後の一致を妨げないため、消去は単調に「可能な位置を増やす」だけです。

## 解法方針

開始位置 `p` が可能である条件は、全ての `j=0..M-1` について

```text
S[p+j] == T[j] または S[p+j] == '#'
```

です。最初に全開始位置を判定し、可能なものをキューへ入れます。キューから `p` を取り出したら区間 `[p,p+M)` を `#` に変えます。

この変更で可能性が変わるのは、変更区間と重なる開始位置だけです。

```text
消した区間 [p, p+M)
影響を受ける開始位置: p-M+1 .. p+M-1
```

この近傍だけを再判定し、新たに可能ならキューへ入れます。最後に全て `#` なら `Yes` です。

## tips

### 実装

開始位置は `0..N-M` です。キューへ入れた位置は `queued` / `used` で管理し、同じ位置を何度も処理しません。

再判定範囲は配列外を切り詰めます。文字を `#` に代入しても、すでに `#` のときは問題ありません。

### よくある誤り

- `#` を不一致として扱う。消去済みの文字はどの文字にも一致します。
- 消去後に全開始位置を毎回再判定する。重なる位置だけで十分です。
- 一度不可能だった位置を二度と見る必要がないとする。隣の消去で可能になります。

### 計算量

各開始位置の判定は長さ `M`、各位置は高々定数回の近傍処理で調べられるため、概ね `O(NM)`、メモリ `O(N)` です。
