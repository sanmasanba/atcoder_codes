# 071

## 問題リンク

[ABC236 F - Spices](https://atcoder.jp/contests/abc236/tasks/abc236_f)

## キーワード

XOR の表現可能性は、線形基底へ独立なベクトルだけを加える

## 何に着目するか

各スパイスの番号は `N` ビットベクトルとみなせ、混合は XOR です。全種類を作るには、選んだベクトルが XOR 空間全体を張る、すなわちランク `N` の線形基底になる必要があります。

独立なベクトルの集合はマトロイドを成すため、コストの小さい順に「独立性を壊さないものだけ採用」する貪欲法が最小コスト基底を作ります。

## 解法方針

空でない各部分集合マスク `x` とコスト `C[x]` を、コスト昇順にソートします。XOR 線形基底で `x` を既存基底から消去し、最後に 0 にならなければ独立です。そのとき採用してコストを足します。

```text
x を最高ビットから基底で XOR 消去
  0 になる     → 既存ベクトルの XOR で表せる。不採用
  0 にならない → 独立。基底へ追加
```

基底は各最高ビットにつき高々一ベクトルなので、長さ `N` の配列で持てます。`N` 本採用した時点で全空間を張るため終了できます。

|候補|基底で表せるか|処理|
|---|---|---|
|`x`|はい|捨てる|
|`x`|いいえ|採用・基底に追加|

## tips

### 実装

`basis[b]` を最高ビット `b` が立つ基底ベクトルとします。`x` の最高ビットから見て `basis[b]` があれば XOR し、なければ `basis[b]=x` として採用します。

コスト合計は 64 bit 整数で持ちます。候補数は `2^N-1` なので、入力順のマスクとコストを取り違えないようにします。

### よくある誤り

- XOR の独立性を通常の集合の重複判定で済ませる。複数の既存ベクトルの XOR で表せる場合もあります。
- 高コストから採用する。求めるのは最小コスト基底です。
- 基底ランクが `N` になる前に終了する。全種類を作るには全次元が必要です。

### 計算量

候補数を `V=2^N-1` とすると、ソート `O(V log V)`、各消去 `O(N)` で合計 `O(V(log V+N))`、メモリは `O(V+N)` です。
