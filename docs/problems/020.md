# 020

## 問題リンク

[問題を開く](https://atcoder.jp/contests/abc263/tasks/abc263_e)

## 何に着目するか

期待値の遷移先が連続区間です。区間和を毎回求め直すと遅くなります。

## 解法方針

ゴールから逆向きに期待値を計算します。次の位置の期待値の区間和を累積和で持てば、各位置の式を定数時間で評価できます。

## 実装要点

期待値は法の下で扱い、区間長の逆元を掛けます。累積和は逆順で更新すると管理しやすくなります。

## 計算量

`O(N)` 時間・空間です。

## 典型・関連問題

典型は **期待値DPの累積和高速化** です。関連: [ABC280 E](https://atcoder.jp/contests/abc280/tasks/abc280_e)
