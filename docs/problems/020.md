# 020

## 問題リンク

[ABC263 E - Sugoroku 3](https://atcoder.jp/contests/abc263/tasks/abc263_e)

## キーワード

期待値 DP の遷移先が区間なら、累積和で平均を高速化する

## 何に着目するか

あるマスからは連続する複数の次マスへ等確率で進みます。期待値の遷移先を1つずつ足すと全体で二乗時間になります。ゴール側から期待値を決めれば、必要なのは次の期待値の連続区間の和です。

## 解法方針

`E[i]` をマス `i` からゴールまでに必要な操作回数の期待値とします。`A[i]` 個先まで均等に進むなら、

```text
E[i] = 1 + (E[i+1] + E[i+2] + ... + E[i+A[i]]) / A[i]
```

です。`suffix[i] = E[i] + E[i+1] + ...` を持てば、区間和は `suffix[i+1] - suffix[i+A[i]+1]` で定数時間に求められます。

| 求める値 | 使う値 |
| --- | --- |
| `E[i]` | 次マスの期待値の区間和 |
| 区間和 | 逆順累積和 `suffix` |
| 割り算 | `A[i]` の法逆元 |

ゴールでは `E[N]=0` です。`i=N-1` から逆順に計算します。

## tips

### 実装

法 `998244353` の下で計算するため、`/ A[i]` は `pow(A[i], MOD-2, MOD)` を掛けます。期待値を決めた直後に `suffix[i]` を更新します。

### よくある誤り

- 現在マスを平均に含める: 遷移先は `i+1` からです。
- 操作1回分の `+1` を忘れる: どの遷移でも最初に1回操作します。
- 累積和を順方向に作る: 未来側の区間を使うため逆順が自然です。

### 計算量

各マスを定数回処理するので `O(N)` 時間・`O(N)` 空間です。
