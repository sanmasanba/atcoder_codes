# 018

## 問題リンク

[ABC258 E - Packing Potatoes](https://atcoder.jp/contests/abc258/tasks/abc258_e)

## キーワード

反復回数が大きい決定的遷移は、一回分を作ってダブリングで飛ばす

## 何に着目するか

箱詰めの開始位置が同じなら、選ぶ個数と次の開始位置は常に同じです。クエリ番号が大きくても、まず「開始位置から1回操作した結果」を前計算すれば、以後は状態遷移を何回も適用する問題になります。

## 解法方針

各開始位置 `i` について、重さの和が `X` 以上になる最短の連続区間を尺取り法で求めます。その長さを `count[i]`、終端の次の位置を `next[i]` とします。

| 前処理 | 得られるもの |
| --- | --- |
| 尺取り法 | `count[i]`, `next[i]` |
| ダブリング | `2^k` 回後の開始位置 |
| クエリ | `K-1` 回遷移した位置の `count` |

配列は円環なので2倍にして扱います。`X` が全体和より大きい部分は、先に何周分かを確定させ、残りだけを尺取り法で求めます。

## tips

### 実装

全体和を `total` とし、`q, r = divmod(X, total)` とします。各開始位置で必ず `q*N` 個を選び、和 `r` を満たす追加部分を尺取りで求めます。ダブリングの基底は `jump[0][i]=next[i]` です。

### よくある誤り

- `K` 回ではなく `K-1` 回遷移する: 1番目の箱詰めは開始位置0そのものです。
- 円環を1周分だけで尺取りする: 区間が末尾をまたぎます。
- `X` が全体和を超える場合を無視する: 何周しても必要な個数が増えます。

### 計算量

尺取り法は `O(N)`、ダブリング表は `O(N log Q)`、各クエリは `O(log Q)` です。
