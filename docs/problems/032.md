# 032

## 問題リンク

[ABC266 F - Well-defined Path Queries on a Namori](https://atcoder.jp/contests/abc266/tasks/abc266_f)

## キーワード

基数環グラフは、葉を削って閉路と各付属木に分ける

## 何に着目するか

辺数と頂点数が等しい連結グラフには、閉路がちょうど一つあります。この閉路上の二頂点間には時計回り・反時計回りの二経路があり得ます。一方、閉路から伸びる木の内部では経路は一意です。

二頂点が「同じ閉路頂点から生えた木」に属するとき、かつそのときに限り、二頂点間の単純パスは一意になります。

## 解法方針

まず次数 1 の頂点をキューに入れ、取り除きます。取り除いた頂点の隣の次数を 1 減らし、新たに次数 1 になればキューへ入れます。

```text
木の葉 → 葉を剥がす → ... → 最後に残る頂点 = 唯一の閉路
```

葉をすべて剥がした後に残る頂点が閉路です。各閉路頂点を別々の根として DFS/BFS を始め、閉路外の頂点へ同じ根 ID を伝播します。

|二頂点 `x,y` の root ID|パス数|答え|
|---|---|---|
|同じ|木の中なので一意|`Yes`|
|異なる|閉路を通る二方向がある|`No`|

閉路頂点自身にも、それぞれ固有の ID を割り当てます。これにより異なる閉路頂点同士は確実に `No` になります。

## tips

### 実装

`on_cycle[v]` を「葉剥がし後に残ったか」で作ります。閉路頂点からの DFS/BFS では、ほかの閉路頂点へは進まないようにします。訪問済み頂点に対して `root[to]=root[v]` を保存します。

多始点 BFS として、すべての閉路頂点を最初にキューへ入れても実装できます。この場合、初期時点で `root[v]=v` としておきます。

### よくある誤り

- 次数 1 を一度だけ消す。新しい葉が次々にできるため、キューで連鎖的に処理します。
- 閉路頂点をすべて同じ root ID にする。異なる閉路頂点間には通常二つの単純パスがあります。
- 閉路頂点から別の閉路頂点にも DFS を伸ばす。木の分類が混ざります。

### 計算量

葉剥がし、分類、クエリ処理はそれぞれ線形です。全体で `O(N+Q)`、メモリは `O(N)` です。
