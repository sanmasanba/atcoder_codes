# 053

## 問題リンク

[ABC276 F - Double Chance](https://atcoder.jp/contests/abc276/tasks/abc276_f)

## キーワード

逐次追加する期待値は、新要素が作る寄与だけで更新する

## 何に着目するか

長さ `i` の prefix から独立に二つ選んだときの `max` の期待値は、全ての順序付き対 `(p,q)` の `max(A[p],A[q])` の和を `i²` で割ったものです。

`A[i]` を追加したときに新しく増えるのは、新要素を少なくとも一方に含む対だけです。過去要素との `max` の和を高速に求められれば、分子を逐次更新できます。

## 解法方針

過去要素について、値 `x=A[i]` 以下の個数 `cnt_le` と和 `sum_le`、全体和 `sum_all` を Fenwick Tree で求めます。

```text
Σ_j max(x, A[j])
= x × cnt_le + (sum_all - sum_le)
```

順序付き対の総和を `S` とすると、新要素を含む対の寄与は、過去要素との組が左右二通り、`(i,i)` が一通りです。

```text
S_new = S_old + 2 × Σ_j max(x, A[j]) + x
answer = S_new / i²
```

|追加される対|寄与|
|---|---|
|`(i,j), (j,i)`（過去 `j`）|各 `max(x,A[j])`、二通り|
|`(i,i)`|`x`|

値は座標圧縮し、個数用と和用の二本の Fenwick Tree を持ちます。

## tips

### 実装

Fenwick Tree へ `x` を追加するのは、上の寄与を計算した後です。`i` は 1-indexed の prefix 長として扱い、最後に `S * inv(i*i)` を出力します。

法が素数なので、`i²` の逆元は累乗で求めるか、`inv[i]` を前計算します。`S`・BIT の和も法で管理します。

### よくある誤り

- 非順序付き対として二倍を忘れる。選ぶ二つの添字は順序付きです。
- `(i,i)` の `x` を足し忘れる。
- `x` 以下と `x` より大きいの境界を取り違える。同値の要素との `max` は `x` です。

### 計算量

各要素で Fenwick Tree の取得・更新を定数回行います。時間 `O(N log N)`、メモリ `O(N)` です。
