# 116

## 問題リンク

[ABC334 E - Christmas Color Grid 1](https://atcoder.jp/contests/abc334/tasks/abc334_e)

## キーワード

追加する白マスが接続する異なる赤成分数だけで成分数の変化を決める

## 何に着目するか

白マスを一つ赤へ変えると、新しい赤成分が一つ生まれます。ただし隣接する赤成分が `k` 個あればそれらとつながって一つに合流するため、全体の成分数変化は `1-k` です。

既存の赤成分を Union-Find で根 ID にすれば、隣接成分の種類数 `k` を数えられます。

## 解法方針

まず隣接する赤マスを Union-Find で併合し、既存の成分数を `C` とします。白マス `v` ごとに、四近傍の赤マスの根を `set` へ入れ、そのサイズを `k` とします。

```text
追加後の成分数 = C + 1 - k
```

|隣接する異なる赤成分 k|変化|追加後|
|---:|---:|---:|
|0|`+1`|`C+1`|
|1|`0`|`C`|
|2|`-1`|`C-1`|
|3,4|さらに合流|`C+1-k`|

全白マスについてこの値を足し、白マス数で割ったものが期待値です。法上では逆元を掛けます。

## tips

### 実装

赤マスだけを ID 化しても、グリッド位置をそのまま `r*W+c` として Union-Find にしても構いません。既存成分数は赤マスの根の種類数を数えるか、併合成功時に減らします。

白マスの隣接根は最大 4 個なので、小さな `set` で十分です。

### よくある誤り

- 隣接する赤マス数をそのまま k とする。同じ既存成分の複数マスは一成分です。
- `C-k` とする。新しい赤マス自身の一成分を加える `+1` が必要です。
- 白マス数で普通の整数除算をする。法上の逆元を使います。

### 計算量

各マス・隣接関係を定数回処理するので、時間 `O(HWα(HW))`、メモリ `O(HW)` です。
