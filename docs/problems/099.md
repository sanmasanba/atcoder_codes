# 099

## 問題リンク

[ABC317 D - President](https://atcoder.jp/contests/abc317/tasks/abc317_d)

## キーワード

目標以上の価値を最小コストで得るなら、価値を軸にナップサック DP する

## 何に着目するか

負けている選挙区を逆転させると、必要な説得人数というコストを払い、その区の議席数 `Z` を得ます。どの選挙区を逆転させるかは 0/1 ナップサックそのものです。

## 解法方針

最初から勝っている区（`X>Y`）の議席数を `base` に足します。負け／同点の区について、逆転に必要な人数を

```text
cost = floor((X+Y)/2) + 1 - X
```

とします。逆転すると議席 `Z` を得ます。

全議席和を `S` とすると、必要な最終議席数は `need=floor(S/2)+1` です。追加で必要なのは `max(0,need-base)` 議席です。

`dp[z]` を「追加議席を少なくとも `z` 得る最小コスト」として、各選挙区を 0/1 で遷移させます。

```text
dp[min(total, z+Z)]
  = min(dp[...], dp[z] + cost)
```

|区の状況|処理|
|---|---|
|`X>Y`|議席を初期値へ加算|
|`X≤Y`|`cost` で議席 `Z` を買える 0/1 品物|

## tips

### 実装

`total=sum(Z)` までの配列を `INF` で初期化し、`dp[0]=0` とします。一つの区を複数回使わないため、議席数のループは大きい方から回すか、新配列を使います。

追加議席が必要ない場合は答え 0 です。

### よくある誤り

- 逆転コストを `(Y-X+1)/2` の丸めで誤る。最終的に `X+t > Y-t` を満たす最小整数です。
- 既に勝つ区も品物へ入れる。コスト 0 で必ず得られる議席です。
- ちょうど半数を目標にする。必要なのは過半数です。

### 計算量

`S=sum(Z)` とすると、時間 `O(NS)`、メモリ `O(S)` です。
