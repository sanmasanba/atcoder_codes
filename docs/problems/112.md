# 112

## 問題リンク

[ABC330 F - Minimize Bounding Square](https://atcoder.jp/contests/abc330/tasks/abc330_f)

## キーワード

答えを二分探索し、固定辺長の区間へ座標を入れる最小移動量を二軸で足す

## 何に着目するか

一辺 `S` の正方形に全点を入れられるかは、x 軸・y 軸で独立です。各軸では長さ S の区間 `[l,l+S]` を選び、外側の座標だけを端へ動かす最小コストを求めます。S が大きいほど必要コストは減るため二分探索できます。

## 解法方針

一軸の座標 `a` に対するコストは

```text
f(l) = Σ max(0, l-a[i]) + max(0, a[i]-(l+S))
```

です。これは凸関数で、`2N` 個の端点 `a[i]` と `a[i]-S` の中央値付近で最小になります。ソートと prefix 和を使えば、任意の l の `f(l)` を `O(log N)` で計算できます。

```text
a < l       → l まで右へ動かす
l ≤ a ≤ l+S → 動かさない
a > l+S     → l+S まで左へ動かす
```

x 座標と y 座標でそれぞれ最小コストを求め、和が移動可能回数 `K` 以下なら辺長 S は可能です。これを最小の S まで二分探索します。

## tips

### 実装

各軸の座標をソートし prefix 和を作ります。候補 l は `a[i]` または `a[i]-S` の中央値から得られ、二分探索または端点列の中央値で求められます。

コスト・二分探索の上界は大きくなるので 64 bit 整数を使います。

### よくある誤り

- 正方形の左下を二次元で全探索する。x と y は独立です。
- 区間内の点にも移動コストを払う。コストは 0 です。
- 固定 S の最適区間を min/max だけで決める。中央の点とのトレードオフがあるため中央値を使います。

### 計算量

ソート後、各判定を `O(log N)` 程度で行い、辺長の二分探索を含めて `O(N log N + log C log N)` 程度です。
