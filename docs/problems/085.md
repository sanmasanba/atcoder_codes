# 085

## 問題リンク

[ABC282 D - Make Bipartite 2](https://atcoder.jp/contests/abc282/tasks/abc282_d)

## キーワード

二部グラフへの辺追加は、連結成分ごとの二色の個数から数える

## 何に着目するか

グラフがすでに二部グラフでなければ、辺を追加して二部グラフにすることはできません。二部グラフなら、追加後も二部性を保つには異なる色の頂点を結ぶ必要があります。

連結成分ごとの二色分けをすると、成分内・成分間の追加可能な辺を数えられます。

## 解法方針

各連結成分を BFS/DFS で 0/1 に彩色し、色数を `(c0,c1)` とします。同じ色の辺を見つけたら答えは 0 です。

追加可能な頂点対は、全ての異色対から既存辺を除いた数です。成分内では `c0*c1` 対、成分間では、二部の色をどちら向きに対応させてもよいので全ての頂点対が追加可能です。

実装しやすい数え方は、成分を順に処理し、既に見た頂点数を `seen` として

```text
answer += c0*c1        # 同じ成分内の異色対
answer += size*seen    # 以前の別成分との全頂点対
seen += size
```

最後に、すでに存在する `M` 本の辺を引きます。

|頂点対|追加後に二部性を保てるか|
|---|---|
|同一成分・同色|不可|
|同一成分・異色|可能（ただし既存辺は除く）|
|別成分|可能|

## tips

### 実装

未訪問を `-1`、色を 0/1 で持ちます。辺をたどり、未訪問なら反対色、既訪問で同色なら即 `0` を出力します。

`answer` は `O(N²)` まで大きくなるので 64 bit 整数です。

### よくある誤り

- 成分間の辺を数えない。別成分は色の反転を自由に選べるので任意の頂点対を結べます。
- `c0*c1` だけを答えにし、既存辺を引かない。追加できるのは存在しない辺だけです。
- 同色辺を発見してもその成分だけを無視する。グラフ全体で不可能です。

### 計算量

全頂点・全辺を一度ずつ見るので、時間 `O(N+M)`、メモリ `O(N+M)` です。
