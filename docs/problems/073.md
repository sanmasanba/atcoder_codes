# 073

## 問題リンク

[ABC244 E - King Bombee](https://atcoder.jp/contests/abc244/tasks/abc244_e)

## キーワード

訪問回数の偶奇だけが必要なら、状態に 1 ビットだけ追加する

## 何に着目するか

必要なのは特定頂点 `X` を何回通ったかの偶奇だけです。正確な訪問回数を持つ必要はありません。頂点と偶奇 `0/1` を組にすると、長さ `K` の経路数を DP できます。

## 解法方針

`dp[step][v][p]` を「`step` 手後に頂点 `v` にいて、`X` の訪問回数の偶奇が `p` である経路数」とします。辺 `v→to` をたどるとき、到着先が `X` なら偶奇を反転します。

```text
next_parity = parity XOR (to == X)
```

|到着先|偶奇の変化|
|---|---|
|`to != X`|そのまま|
|`to == X`|`0↔1`|

始点 `S` 自身が `X` なら初期偶奇は 1 です。`K` 手後に `T` にいて偶奇 0 の値が答えです。

## tips

### 実装

二層の `N×2` 配列を交互に使えば、手数次元を持たずに済みます。無向グラフの各辺は両方向へ遷移させます。

### よくある誤り

- `X` から出るときに偶奇を反転する。訪問は到着時（始点も含む）として数えます。
- 始点が `X` の初期偶奇を 0 にする。
- `K` 手未満で `T` に着いた経路を答えへ加える。ちょうど `K` 手です。

### 計算量

各手で全辺を一度ずつ使うため、時間 `O(KM)`、二層 DP のメモリは `O(N)` です。
