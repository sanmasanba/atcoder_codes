# 080

## 問題リンク

[ABC272 E - Add and Mex](https://atcoder.jp/contests/abc272/tasks/abc272_e)

## キーワード

時刻ごとの値を問うなら、各要素が有効な時刻範囲を逆算して配る

## 何に着目するか

時刻 `m` における要素 `i` の値は `A[i] + i*m` です。mex は `0..N` のどこかに必ずあるため、`0..N` の外にある値は無視して構いません。

各要素が有効範囲へ入る時刻だけを求め、その時刻のバケットへ値を配ると、全時刻を要素ごとに全走査せずに済みます。

## 解法方針

添字を 1-indexed とし、要素 `i` の値を

```text
v(m) = A[i] + i*m
```

とします。これが `0≤v(m)≤N` となる `m` の範囲は不等式から求められます。

```text
start = max(1, ceil(-A[i] / i))
end   = min(M, floor((N-A[i]) / i))
```

`start≤end` のとき、その全 `m` について値 `v(m)` を `bucket[m]` へ追加します。

|要素 i|有効な時刻 m|bucket[m] へ入れる値|
|---|---|---|
|`i`|`start..end`|`A[i]+i*m`|

各 `bucket[m]` には、その時刻で mex の候補になり得る値だけが集まります。値を出現印配列へ印し、`0` から最初に印のない値を探せば mex です。

## tips

### 実装

負の整数の切り上げ除算に注意します。Python では `ceil(-a/i)` を `(-a + i - 1)//i` と単純に書けるのは分子が非負の場合だけなので、`start=max(1, (-A[i]+i-1)//i)` の意味を確認します。`A[i]` が正なら start は 1 でよいので分けても安全です。

各時刻の mex は高々 `N` なので、バケットの値だけを `set` にするか、時刻ごとに印配列を使います。

### よくある誤り

- `v>N` の値もバケットへ入れる。mex には影響せず、計算量だけが増えます。
- `m=0` を入れる。問題の時刻は `1..M` です。
- 負の値に対する整数除算の丸めを言語仕様のまま使い、開始時刻を一つずらす。

### 計算量

要素 `i` が追加される回数は高々 `O(N/i)` 程度です。総追加数は調和級数的に `O(N log N)`、mex 計算を含めて全体 `O(N log N)` 程度、メモリも総追加数に比例します。
