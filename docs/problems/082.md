# 082

## 問題リンク

[ABC274 E - Booster](https://atcoder.jp/contests/abc274/tasks/abc274_e)

## キーワード

訪問で移動能力が変わる巡回は、訪問集合から能力を復元する bit DP にする

## 何に着目するか

全ての町を訪れて原点へ戻る必要があります。加速器は訪問必須ではありませんが、一度訪れると以後の速度が 2 倍になります。どの地点を訪れたかが、現在位置だけでなく速度も決めます。

訪問済み集合と現在地を状態にした bit DP を使います。

## 解法方針

町 `0..N-1` と加速器 `N..N+M-1` をまとめて `P=N+M` 個の地点として扱います。`dp[mask][v]` を「`mask` の地点を訪問済みで、現在 `v` にいる最短時間」とします。

`mask` に含まれる加速器数を `b` とすると速度は `2^b` です。未訪問地点 `to` への遷移は

```text
dp[mask | (1<<to)][to]
  = min(..., dp[mask][v] + dist(v,to) / 2^b)
```

```text
訪問集合 mask
  ├ 町: 最終的に全て必要
  └ 加速器: 任意、含まれる数が速度を決める
```

全ての町ビットを含む `mask` ごとに、現在地 `v` から原点へ戻る時間 `dist(v,origin)/2^b` を足します。その最小が答えです。加速器を取った後の帰還も速くなるため、必ず戻る遷移を評価します。

## tips

### 実装

原点は訪問集合に含めず、初期状態を `dp[0][origin]=0` としても、原点を地点 0 として別扱いしても構いません。実装の添字を統一します。

加速器数は `popcount(mask >> N)` で求められます。`2^b` は `1<<b` で整数として出し、距離計算は `float` / `double` を使います。

### よくある誤り

- 全ての加速器を訪れる必要があるとする。加速器は任意で、遠回りなら取らないほうがよいです。
- 全町訪問時点の `dp` を答えにする。原点へ戻る時間が必要です。
- 加速器へ移動する前の速度ではなく、到着後の速度でその辺を割る。速度上昇は到着後からです。

### 計算量

地点数を `P=N+M` とすると、状態遷移は `O(2^P P^2)`、メモリは `O(2^P P)` です。小さい `P` に対する典型的な巡回 bit DP です。
