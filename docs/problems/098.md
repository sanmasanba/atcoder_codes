# 098

## 問題リンク

[ABC315 D - Magical Cookies](https://atcoder.jp/contests/abc315/tasks/abc315_d)

## キーワード

削除で新たな削除候補が生まれるなら、キューで連鎖処理する

## 何に着目するか

削除できる行・列は、残っているマスが二個以上あり、その色が一種類だけのものです。一つ削除すると交差する列／行の色数が減り、新たに削除可能になることがあります。単調に削除だけが起こるためキュー処理です。

## 解法方針

各行・列について、残っている各文字の個数と残りマス数を管理します。

```text
削除可能 ⇔ 残りマス数 ≥ 2 かつ 出現文字種数 = 1
```

初めに条件を満たす全行・列をキューへ入れます。行を削除する場合、まだ残っている各列の対応文字カウントを一減らします。文字種数が変化して新たに条件を満たした列をキューへ入れます。列を削除する場合も対称です。

```text
削除行 → 各列のカウント更新 → 新しい単色列をキューへ
削除列 → 各行のカウント更新 → 新しい単色行をキューへ
```

最後に残った行数×列数が答えです。

## tips

### 実装

`alive_row`, `alive_col` を持ち、キューから取り出した時点で既に削除済みなら飛ばします。行を削除するときは生存列だけ、列を削除するときは生存行だけを更新します。

文字種数は 26 個のカウント配列を走査してもよく、カウントが 0 になった瞬間だけ種類数を減らしても構いません。

### よくある誤り

- 残りマス数 1 の行・列も削除する。問題の条件は二個以上です。
- 初期に一回だけ判定して終える。削除で新しい単色行・列が生まれます。
- 削除済みの行／列のマスもカウントから再び引く。

### 計算量

各マスは、その行または列が削除されるとき高々一度更新されます。時間 `O(HW)`、メモリ `O(HW+26(H+W))` です。
