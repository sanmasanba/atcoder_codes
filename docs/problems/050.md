# 050

## 問題リンク

[ABC246 D - 2-variable Function](https://atcoder.jp/contests/abc246/tasks/abc246_d)

## キーワード

二変数で単調な関数は、片方ずつ動かす二ポインタで探索する

## 何に着目するか

`f(a,b)=a³+a²b+ab²+b³` は、`a,b≥0` でどちらの変数を増やしても減りません。各 `a` について条件 `f(a,b)≥N` を満たす最小の `b` を探したいとき、`a` が増えれば必要な `b` は増えません。

そのため、`b` を毎回 0 から探し直す必要はなく、大きい端から一方向にだけ動かせます。

## 解法方針

`a=0` から増やし、`b=10^6` から始めます。`f(a,b)≥N` である間は `b` を減らし、条件を満たさなくなる直前の `(a,b)` を候補として答えに使います。

```text
a を増やす → f は増える
              ↓
同じ N を満たす最小 b は、前回より右へ戻らない
```

実装では「`b≥0` かつ `f(a,b)≥N` の間 `b--`」とし、ループを抜けた後の `b+1` を候補にします。あるいは候補評価を減算前に行う書き方でも構いません。

|操作|`a`|`b`|
|---|---:|---:|
|初期|`0`|大きい値|
|`f(a,b)` が十分大きい|固定|`b--`|
|次の `a`|`a++`|前回の `b` のまま|

## tips

### 実装

`a,b` は `0..10^6` を調べれば十分です。`10^6` の三乗は `10^18` なので、C++ では `long long` を使い、積の途中でも桁あふれしないことを確認します。

`b` が `-1` になった場合は、それ以上内側へ減らせません。`a` についてのループは続けて構いません。

### よくある誤り

- `a` ごとに `b=10^6` へ戻す。二重ループになり遅くなります。
- 条件を満たさなくなった `b` を候補にする。最後に満たしていたのは `b+1` です。
- `a,b` を正の整数だけにする。0 も許されます。

### 計算量

`a` は高々 `10^6+1` 回、`b` 全体も高々 `10^6+1` 回しか減りません。時間は `O(10^6)`、追加メモリは `O(1)` です。
