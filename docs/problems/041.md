# 041

## 問題リンク

[ABC288 D - Range Add Query](https://atcoder.jp/contests/abc288/tasks/abc288_d)

## キーワード

固定長の区間加算は、長さを法とする剰余類ごとの不変量を見る

## 何に着目するか

長さ `K` の連続区間へ同じ値 `c` を足す操作では、添字を `K` で割った各余り類にちょうど一つずつ `c` が足されます。したがって、余り類ごとの要素和の**差**は操作で変わりません。

全要素を 0 にできるなら、最終的には各余り類の和がすべて 0 です。よって、開始時点で余り類ごとの和がすべて等しいことが必要です。そしてこれは十分でもあります。

## 解法方針

クエリ区間 `[l,r]` の中で、各余り `t=0..K-1` について次を求めます。

```text
sum[t] = Σ A[i]   (l ≤ i ≤ r, i mod K = t)
```

この `K` 個の値がすべて同じなら `Yes`、一つでも異なれば `No` です。

|操作前の余り類和|長さ K の区間へ `c` を加える|操作後|
|---|---|---|
|`(s0, s1, ..., sK-1)`|各類に一回ずつ `+c`|`(s0+c, s1+c, ..., sK-1+c)`|

等しければ、左から順に「先頭要素を 0 にする」操作を行えます。最後に残る `K-1` 個についても余り類和が等しいという不変量から全て 0 になります。これが十分性です。

前処理として、各余り類ごとに元配列上の累積和を持ちます。クエリでは余り `t` ごとに区間和を一回取得するだけです。

## tips

### 実装

`prefix[t][i+1]` を「先頭 `i+1` 個のうち添字 `mod K=t` の値の和」とします。あるいは、各余り類の位置だけを抜き出した配列を作っても構いません。

クエリごとに `K` 個の和を求め、最初の和と全て比較します。`K≤10` なので `O(K)` 判定で十分です。入力が 1-indexed か 0-indexed かに合わせて余りを統一します。

### よくある誤り

- 区間全体の和が 0 かだけを見る。必要なのは余り類 `K` 個の和が互いに等しいことです。
- 各操作で一つの余り類だけが増えると考える。長さがちょうど `K` なので全ての余り類が一回ずつ現れます。
- 区間の左端をずらしたときの余りをローカル添字で取る。元配列の添字による余りでよく、`K` 類の順序が巡回するだけです。

### 計算量

前処理は `O(NK)`（各余り類の累積配列を作る実装）または `O(N)`、各クエリは `O(K)` です。メモリは `O(NK)` または `O(N)` です。
