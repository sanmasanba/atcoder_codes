# 016

## 問題リンク

[ABC256 E - Takahashi's Anguish](https://atcoder.jp/contests/abc256/tasks/abc256_e)

## キーワード

機能グラフの閉路を取り出すなら、入次数 0 を連鎖的に削る

## 何に着目するか

各頂点から出る辺がちょうど1本のグラフでは、どの連結成分にも閉路が1つだけあります。閉路の外の頂点は、誰かが止められても問題になりません。必ず1人を選ぶ必要があるのは閉路上だけであることに注目します。

## 解法方針

入次数が0の頂点は閉路に含まれません。その頂点を取り除くと、行き先の入次数が1つ減ります。新たに入次数0になった頂点も同様に取り除く操作を繰り返すと、最後に残る頂点はすべて閉路上です。

| 頂点の状態 | 処理 |
| --- | --- |
| 入次数0 | キューへ入れて削除 |
| 削除された頂点の行き先 | 入次数を1減らす |
| 最後まで残った頂点 | 閉路上。各閉路で最小コストを足す |

残った頂点を未訪問からたどり、1つの閉路ごとにコストの最小値を答えへ加えます。

## tips

### 実装

最初に全頂点の入次数を数え、入次数0をキューへ入れます。削除済み配列を持つか、入次数を負の値にして区別します。閉路探索では、すでに数えた頂点を再度たどらないよう訪問済みにします。

### よくある誤り

- 各連結成分の全頂点で最小値を取る: 必要なのは閉路上の最小値です。
- 入次数0を1回だけ消す: 削除によって新しい入次数0が生まれます。
- 残った各頂点のコストを全て足す: 閉路ごとに選ぶのは1頂点です。

### 計算量

各頂点・各辺を高々定数回処理するため、`O(N)` 時間・`O(N)` 空間です。
