# 095

## 問題リンク

[ABC306 E - Best Performances](https://atcoder.jp/contests/abc306/tasks/abc306_e)

## キーワード

動的な上位 K 個の和は、二集合に分けて境界を調整する

## 何に着目するか

一点更新のたびに「大きい方から K 個の和」を求めます。毎回ソートすると遅いですが、上位 K 個の集合 `high` と残り `low` を分け、`high` の和だけを維持すればよいです。

## 解法方針

二つの多重集合を次の不変条件で持ちます。

|集合|内容|保持する値|
|---|---|---|
|`high`|大きい方からちょうど K 個|`sum_high`|
|`low`|残り|最大要素|

常に `min(high) ≥ max(low)` と `|high|=K` にします。更新 `(i,x)` では、古い `A[i]` を入っている側から削除し、新しい `x` を適切な側へ追加します。その後、サイズと境界を整えます。

```text
low  | high
小さい値 ... | ... 大きい K 個
max(low) ≤ min(high)
```

整える操作は、`high` が足りなければ `low` の最大を移す、`high` が多ければ `high` の最小を戻す、境界が逆転したら両端を交換する、の三つです。各クエリの答えは `sum_high` です。

## tips

### 実装

C++ なら `multiset` 二個が分かりやすいです。Python では削除可能な heap 二個より、座標圧縮した値に対して Fenwick Tree を二本（個数・値和）持つと安定します。K 番目に大きい値を二分探索し、上位和を復元できます。

初期状態の全値は 0 です。重複する 0 を多重集合として正しく入れます。

### よくある誤り

- 新値を入れる前に古値を削除しない。同じ添字の値が二重に残ります。
- `high` の和を、`low` との交換時に更新し忘れる。
- 集合ではなく重複を消す set を使う。同じ点数の人が複数います。

### 計算量

各更新で定数回の集合操作をするため `O(log N)`、メモリは `O(N)` です。
