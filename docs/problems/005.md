# 005

## 問題リンク

[ABC235 D - Multiply and Rotate](https://atcoder.jp/contests/abc235/tasks/abc235_d)

## キーワード

操作回数を最小化するときは、状態そのものを頂点にして BFS する

## 何に着目するか

操作回数を最小化したい一方で、各操作のコストはどちらも1です。現在の整数が分かれば、次にできる操作と遷移先が決まります。この「整数を状態、操作を辺」とみなせば、最短手数は幅優先探索で求められます。

## 解法方針

到達し得る整数 `v` を頂点とし、次の2種類の辺を張ります。

| 操作 | 遷移先 | 可能な条件 |
| --- | --- | --- |
| 掛け算 | `v × a` | 桁数の上限以内 |
| 回転 | 末尾の桁を先頭へ移す | `v >= 10` かつ末尾が `0` でない |

すべての辺の重みが同じなので、`1` からBFSを始め、`N` に初めて着いた距離が最小手数です。すでに訪れた値へは再び進まず、距離配列で管理します。

探索範囲を有限にするには、目標 `N` と同じ桁数以下だけを残します。操作で桁数が増えた状態は、その後の掛け算でも回転でも桁数を減らせないため、`N` には到達できません。

回転は、たとえば `1234 → 4123` です。末尾が `0` の `120` を回転すると先頭の0が消えて桁数が変わるため、この操作は使えません。

## tips

### 実装

上限を `limit = 10 ** len(str(N))` とし、`0 <= next < limit` の状態だけをキューへ入れます。回転後の値は文字列操作でも、`(v % 10) * 10^(d-1) + v // 10` の整数演算でも作れます。

### よくある誤り

- 深さ優先探索を使う: 最初に見つけた経路が最短とは限りません。
- 桁数上限を置かない: 掛け算で状態が無限に増えます。
- 末尾が0の回転を許す: 問題の操作条件に反し、状態空間も崩れます。

### 計算量

探索状態数を `V` とすると、各状態から高々2遷移なので `O(V)` 時間・`O(V)` 空間です。ここで `V` は `N` の桁数を `d` として高々 `10^d` 程度です。
